**Neuer Mode C — Desktop Sidebar auf Detail-Route:**
Trigger: `context !== undefined && isExpanded && isOpen` auf Desktop (wir detectieren Desktop in FloatingChat bereits via Tailwind-Breakpoint-Klassen; für JS-gate neuen `useMediaQuery`-Hook oder inline via `window.matchMedia`).

Layout-Unterschied zu B:
- **B (Home-expanded):** `md:w-[35%]` rechte Sidebar, Main-Area shrinkt `md:mr-[35%]` (aktuell in AppShell/GlobalLayout).
- **C (Detail-expanded mit context):** `md:w-[400px]` (fix 400px, nicht 35%), Main-Area shrinkt `md:mr-[400px]`, **und** der Detail-Artikel schrumpft zusätzlich intern von `max-w-2xl` → `max-w-prose` / weiter (Artikel-Anpassung ist Teil von Plan 03, hier nur Sidebar-Breite richtig setzen).

Implementation:

1. **JS-Desktop-Detection** (neuer lokaler State `isDesktop`):
   ```typescript
   const [isDesktop, setIsDesktop] = useState(false)
   useEffect(() => {
     const mq = window.matchMedia('(min-width: 1024px)') // lg-Breakpoint, konsistent mit Maximize-Button-Visibility
     const update = () => setIsDesktop(mq.matches)
     update()
     mq.addEventListener('change', update)
     return () => mq.removeEventListener('change', update)
   }, [])
   ```

2. **Layout-Mode-Derivation**:
   ```typescript
   const isSidebarMode = !!context && isExpanded && isOpen && isDesktop
   ```

3. **Render-Condition anpassen** (Z.405): statt `if (isExpanded && isOpen)` jetzt `if (isExpanded && isOpen)` **beibehalten**, aber innerhalb des Blocks die Klassen konditional machen:
   ```tsx
   className={`fixed top-[77px] bottom-[96px] left-0 right-0
     md:inset-auto md:top-[148px] md:bottom-0
     ${isSidebarMode
       ? 'md:right-0 md:w-[400px]'
       : 'md:right-0 md:w-[35%]'
     }
     flex flex-col rounded-t-2xl md:rounded-l-2xl md:rounded-tr-none …`}
   ```
   (Die bestehende `md:top-[148px]` passt zu Home wo FilterBar existiert; auf Detail-Route gibt es keine FilterBar — siehe Plan 03 für die Top-Offset-Anpassung via CSS-Variable oder zweiter Conditional.)

   **Für 15-02 reicht `md:top-[148px]` stehen zu lassen.** Plan 03 passt das an basierend auf fehlendem FilterBar.

4. **Main-Area-Shrink in GlobalLayout**: Die aktuelle Zeile `${isChatExpanded ? 'md:mr-[35%]' : ''}` auf GlobalLayout.tsx muss konditional auf `chatContext` reagieren:
   ```tsx
   <main className={`flex flex-col flex-1 overflow-hidden transition-all duration-300
     ${isChatExpanded ? (chatContext ? 'lg:mr-[400px]' : 'md:mr-[35%]') : ''}`}>
   ```
   — Home behält `md:mr-[35%]`, Detail bekommt `lg:mr-[400px]` (fix).

5. **Artikel-max-width-Shrink** ist Plan 03 (dort, wo die Detail-Page gerendert wird).
