`app/layout.tsx` bekommt einen conditional Wrapper:

1. `getUser()` bleibt wie bisher (Z.102). Mode ableiten:
   ```typescript
   const user = await getUser()
   const mode: ChatMode = user ? 'member' : 'public'
   ```

2. Pathname-Detection: In Server Components gibt es kein `usePathname`. Workaround: `headers()` auslesen und aus `x-pathname`/`next-url` den Path extrahieren. **Cleaner Approach:** Einen neuen Client-Wrapper `ConditionalGlobalLayout.tsx` bauen der `usePathname` nutzt:

   ```typescript
   'use client'
   import { usePathname } from 'next/navigation'
   import GlobalLayout from './GlobalLayout'

   const BARE_ROUTES = ['/login']

   export function ConditionalGlobalLayout({ mode, children }: Props) {
     const pathname = usePathname()
     const isBare = BARE_ROUTES.some(route => pathname === route || pathname.startsWith(route + '/'))
     if (isBare) return <>{children}</>
     return <GlobalLayout mode={mode}>{children}</GlobalLayout>
   }
   ```

   Platzierung: `components/layout/ConditionalGlobalLayout.tsx`.

3. In `app/layout.tsx`:
   ```tsx
   <ThemeProvider>
     <AuthProvider initialUser={user}>
       <ConditionalGlobalLayout mode={mode}>
         {children}
       </ConditionalGlobalLayout>
     </AuthProvider>
   </ThemeProvider>
   ```

4. **Auth-Callback-Routen prüfen:** `/auth/callback`, `/auth/signout` — haben die eigene Pages oder sind Route-Handler? Falls Pages: in `BARE_ROUTES` aufnehmen (kein Chat sinnvoll auf Callback). Mit `apps/tools-app/app/auth/` kurz prüfen — falls Route-Handler (route.ts), betrifft sie layout.tsx ohnehin nicht.

5. **Impressum/Datenschutz-Routen:** Sind Content-Pages mit eigenem Reading-Layout — der Chat würde dort stören. Optional in BARE_ROUTES, aber Decision liegt nicht explizit gelockt → **pragmatisch BARE_ROUTES = ['/login'] lassen**, Legal-Seiten bekommen GlobalLayout (Chat kann dort mitgezeigt werden, ist hilfreicher als störend; Luca kann das in einem Follow-up adjustieren falls unerwünscht).

Das `pt-24` war Abstand für einen nicht-existenten sticky Header — mit GlobalLayout-Header jetzt obsolet, auf `pt-8` reduzieren.
