# Plan 24-02 — Assessment Types + Scoring

## Objective

Build the core scoring logic: discriminated-union TypeScript types for all 9 question shapes + 9 answer shapes, a pure `scoreAssessment()` function (deterministic, client-safe, no LLM, no I/O), and vitest fixtures proving the same input always yields the same output. This module is the integrity backbone of D-01.

## Context

Per 24-RESEARCH §1 and CONTEXT.md D-01 / D-02 / D-04: scoring must be reproducible, pure math, computed client-side after the last question. Level thresholds must be hard-coded constants (not config). Dimension scores normalized 0-100. Levenshtein for rank widget (W2). Distance-based points for confidence widget (W7).

  <action>
    Create `apps/website/lib/assessment/scoring.ts` exporting:

    1. Constants:
    ```ts
    export const LEVEL_THRESHOLDS: Array<{ min: number; max: number; level: Level; slug: LevelSlug; name: string }> = [
      { min: 0,  max: 5,  level: 1, slug: 'neugieriger',     name: 'Neugieriger' },
      { min: 6,  max: 12, level: 2, slug: 'einsteiger',      name: 'Einsteiger' },
      { min: 13, max: 19, level: 3, slug: 'fortgeschritten', name: 'Fortgeschritten' },
      { min: 20, max: 25, level: 4, slug: 'pro',             name: 'Pro' },
      { min: 26, max: 30, level: 5, slug: 'expert',          name: 'Expert' },
    ]
    ```
    (These thresholds are tuned for 10 questions × 3 maxPoints each = 30. Plan 24-03 commits that distribution; if it deviates, update thresholds in the SAME task and add/adjust fixtures in 24-02-03.)

    2. Per-question scorer — export `function scoreQuestion(question: Question, answer: Answer | undefined): number` that returns earned points (floor 0, ceiling `question.maxPoints`). Implementations:
       - `pick` / `mc`: return `options.find(o => o.id === answer.optionId)?.points ?? 0`
       - `rank`: if `scoring === 'exact'` → `order === correctOrder` ? maxPoints : 0. Else Levenshtein on id arrays → `Math.max(0, maxPoints - distance)`
       - `best-prompt`: same as pick on `options`
       - `side-by-side`: `(choice === correctChoice ? choicePoints : 0) + reasonIds.filter(rid => reasons.find(r=>r.id===rid)?.isCorrect).length * reasonPointPerCorrect` (clamped to maxPoints)
       - `spot`: `passageSegments.find(s => s.id === answer.segmentId)?.isCorrect ? maxPoints : 0`
       - `match`: `Object.entries(pairs).filter(([taskId, toolId]) => correctPairs[taskId] === toolId).length * pointPerCorrect` (clamped to maxPoints)
       - `confidence`: `pointByDistance[Math.abs((answer.step ?? 2) - groundTruthStep)]`
       - `fill`: `blanks.reduce((sum, b) => sum + (b.options.find(o => o.value === selections[b.id])?.isCorrect ? b.pointsIfCorrect : 0), 0)` (clamped to maxPoints)

    3. Levenshtein helper — internal `function levenshteinArr<T>(a: T[], b: T[]): number` as per 24-RESEARCH §1 DP implementation.

    4. `export function scoreAssessment(questions: Question[], answers: Record<string, Answer>): AssessmentResult` — does:
       - Iterate questions, compute earnedPoints per question via scoreQuestion
       - Sum totalPoints, maxPoints
       - Per-dimension sums: for each Dimension, sum earnedPoints where question.dimension matches; sum maxPoints where matches
       - Per-dimension percentage: `Math.round((dimEarned / dimMax) * 100)` (if dimMax === 0, return 0)
       - Determine level: find threshold where `totalPoints` is in `[min, max]`. If `totalPoints` exceeds the highest threshold's max (content shift), return the highest level. If dimensions are zero, return level 1.
       - Return `{ totalPoints, maxPoints, level, slug, skills: { tools, prompting, agents, application, literacy } }`

    5. Helper: `export function getLevelByPoints(points: number): { level: Level; slug: LevelSlug; name: string }` — pure util, derived from LEVEL_THRESHOLDS.

    The function MUST be pure: no `Math.random()`, no `Date.now()`, no `fetch`, no side effects. Same (questions, answers) → same result every call.
  </action>
