<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MDX frontmatter → fs.readFile | Author-controlled MDX-Files, parsed via gray-matter at request time |
| MDX body → React render | gray-matter strips frontmatter; @next/mdx compiles body. Custom remark/rehype plugins are EMPTY (no untrusted-content path) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01-01 | Tampering | `validateArticleFrontmatter` | mitigate | Hard-throw on missing/invalid fields — fail build, not runtime undefined (D-20) |
| T-26-01-02 | Information Disclosure | `readAllFrontmatter` error path | mitigate | Throw with file-path context but not full fs error stack to render output (errors thrown server-side go to logs, page renders generic 500/404) |
| T-26-01-03 | Tampering | MDX body `<script>` injection | mitigate | @next/mdx escapes HTML by default. NO custom remark/rehype plugins added (RESEARCH §2.1 keeps both arrays empty) |
| T-26-01-04 | Spoofing | filename↔slug mismatch | mitigate | Validation throws when `frontmatter.slug !== fileSlug` — prevents author confusion + path-mismatch in sitemap |
| T-26-01-05 | Information Disclosure | `circleUrl` open-redirect | accept | D-14 trust-the-author. URLs are editor-controlled MDX-Frontmatter, no user input path. Documented in README. |
</threat_model>
