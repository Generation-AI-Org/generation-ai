# Plan 22.8-07 — Join Hero-Layout-Refinement

**Status:** ✅ DONE 2026-04-26

## Objective

Refine `/join` so the hero contains the conversion form above the fold on desktop: headline/content on the left, form on the right, mobile stacked. Preserve the existing Phase 23/22.8-06 form behavior instead of rebuilding it.

## Existing Basis

- `/join` currently renders `JoinHeroSection` and `JoinFormSection` as consecutive sections.
- `JoinFormCard` already contains the Phase 22.8-06 status switch, conditional university field, study-field select, sessionStorage draft, server-action submit, and success-card swap.
- This plan is a layout delta only.

## Tasks

1. Add a form slot to `JoinHeroSection` and make the hero a responsive two-column layout when the slot is present.
2. Make `JoinFormSection` support an embedded mode that returns the form/success swap without a nested `<section>`.
3. Wire `JoinClient` to render the embedded form inside the hero instead of rendering a second form section below it.
4. Add compact spacing props to `JoinFormCard` and `JoinSuccessCard` so the existing form fits inside the hero viewport.
5. Verify TypeScript, build, and Join E2E smoke.

## Delta Notes

- Form is embedded in the hero in compact mode on `/join`.
- Desktop uses a left-copy / right-form grid; mobile stacks copy then form with tighter mobile-only spacing.
- Status card helper text uses Sans while labels remain Mono for readability.
- University combobox has 61 fixed Hochschulen/Universitäten plus deterministic `Andere`, live filtering, and an optional detail field.
- Study field `Sonstiges` reveals an optional detail field.
- No new live DB migration: V1 keeps `university` / `study_field` categorical and carries free-text detail via existing `study_program`; Signup V2 also receives detail metadata.

## Acceptance Criteria

- Desktop `/join` has H1 left and form right in the hero viewport.
- Mobile `/join` stacks headline then form.
- Existing form fields, status switch, sessionStorage draft, submit flow, and success-card CTA remain intact.
- No nested `<section>` inside the hero.
- `pnpm --filter @genai/website exec tsc --noEmit` passes.
- `pnpm build` passes.
- Join E2E remains green or only fails on live-service prerequisites already documented.
