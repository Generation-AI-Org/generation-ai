# Plan 22.8-05 — Partner Nav: Split-Button (Click vs Chevron-Dropdown)

## Goal

Aus Call-Feedback 2026-04-25: "Klick auf 'Für Partner' im Header sollte zur `/partner`-Übersicht navigieren, nur das Chevron öffnet das Dropdown." Vorher war der gesamte Trigger ein DropdownMenuTrigger — Klick auf den Text öffnete das Dropdown statt zu navigieren.

## Changes

### Desktop Nav (`header.tsx:84–105`)
- "Für Partner" als `<Link href="/partner">` extrahiert
- Chevron als separater `<DropdownMenuTrigger>` daneben, nur Icon-Button
- `aria-label="Partnertypen-Untermenü öffnen"` am Chevron
- `-ml-1` am Chevron-Button für visuelle Nähe zum Link

### Mobile Nav (`header.tsx:226–267`)
- Analog: "Für Partner" Text als `<Link>` mit `SheetClose`-Wrapper (schließt Sheet beim Tap)
- Chevron als separater Toggle-Button rechts, behält Akkordeon-Verhalten für Submenu
- `flex items-center` Container, Link ist `flex-1`, Button rechts

## Decisions

- Sub-Items (Unternehmen/Stiftungen/Hochschulen/Initiativen) bleiben unverändert — klickbar im Dropdown via `?typ=` Deep-Link
- Mobile Pattern: Tap auf Text schließt Menü + navigiert; Tap auf Chevron toggelt Submenu (kein Sheet-Close)
- `partnerSubItems`-Konstante unverändert wiederverwendet

## Must-Haves

- [x] Desktop: Klick auf "Für Partner" Text → `/partner`
- [x] Desktop: Klick auf Chevron → Dropdown öffnet
- [x] Mobile: Tap auf "Für Partner" → schließt Sheet + navigiert
- [x] Mobile: Tap auf Chevron → Submenu auf/zu, Sheet bleibt offen
- [x] aria-label am Chevron
- [x] Keyboard-Reachability (Tab-Order: Link → Chevron → nächster Link)

## Out-of-Scope

- Sub-Item-Routing (bestehend, nicht angefasst)
- Active-State-Highlighting auf `/partner` (bisher kein Pattern, kein Call-Feedback)
