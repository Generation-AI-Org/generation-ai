# Plan 22.8-01 — DS-Compliance-Audit

> **Read-only Inventur, KEIN Code-Change.** Foundation-Plan vor allem anderen — produziert Decision-Tabelle die Plan 22.8-02 (Token-Fix) füllt.

---

## Objective

Inventur aller Components im Repo (`apps/website/`, `apps/tools-app/`, `packages/ui/`) die

1. hardcoded Farben statt DS-Tokens nutzen (z.B. `bg-green-500` statt `bg-[var(--accent)]`)
2. Hover-States haben die nicht dem canonical Pattern entsprechen (`brand/Generation AI Design System/README.md:89`: `scale(1.015) + Shadow + Border zu accent-ramp-30%`)
3. Light-Mode-inkonsistent sind (Component sieht im Toggle-Light hässlich aus weil hardcoded Dark-Werte)
4. Logo-Sichtbarkeit-Issues haben (zu dunkel auf Hover, fehlender Hover-State auf Brand-Logos)

**Output:** `22.8-01-AUDIT.md` mit Decision-Tabelle pro Finding:

| ID | File | Line | Issue | Severity | Empfehlung |
|---|---|---|---|---|---|
| A-01 | `components/ui/example.tsx` | 42 | `bg-green-500` hardcoded | must | Replace with `bg-[var(--accent)]` |
| ... | ... | ... | ... | ... | ... |

Severity-Skala:
- **must:** Pro-Launch-Blocker (visuell falsch, broken im anderen Theme)
- **should:** Konsistenz-Drift, fixbar in 22.8-02
- **nice:** Cosmetic, kann ins Backlog

---

---

## Scope

**In-scope:**
- Recursive Grep auf `bg-`, `hover:`, `text-`, `border-` mit hardcoded Tailwind-Color-Klassen (z.B. `bg-green-`, `text-red-`, etc.)
- Recursive Grep auf inline `style={{ ... color: '#...' }}` mit Hex-Codes
- Manual review der Hauptkomponenten (Card-Variants, Button-Variants, Section-Wrapper)
- Check Logo-Components: `<Logo />` Varianten — sind alle Hover-States DS-konform?
- Light-Mode-Visual-Check: Toggle aktivieren, manuell durch alle Hauptseiten klicken, Inkonsistenzen markieren

**Out-of-scope:**
- Jeglicher Code-Change (kommt in Plan 22.8-02)
- Refactoring von Component-APIs
- A11y-Audit (eigener Bereich, später)
