# Plan 22.8-04 — Partner Hero: 4 Boxen + Lucide-Icons

## Goal

Above-the-fold-Hierarchie auf `/partner` schärfen: 4 Tab-Boxen (Unternehmen / Stiftungen / Hochschulen / Initiativen) mit Lucide-Icons direkt im Hero, klickbar als Quick-Jump zum Tab-System.

## Context

Aus 22.8-CONTEXT.md D-06: "Partner-Hero größer = Lucide-Icons pro Box + leicht erhöhtes Padding für Icon-Platz + Above-the-fold-Sicherheit. Nicht globale Card-Höhe ändern. Bleibt im Unterseiten-Hero-Pattern (max-w-4xl, --fs-display, LabeledNodes-BG)."

Vor diesem Plan war der Partner-Hero rein textbasiert (Eyebrow + H1 + Subline + Lede). Die 4 Partnertypen waren erst im Tab-System darunter sichtbar — kein visueller Anker above-the-fold.

## Changes

### `partner-hero-section.tsx`
- `onTypChange?: (slug: PartnerTyp) => void` Prop hinzugefügt
- Lucide-Imports: `Building2`, `Landmark`, `GraduationCap`, `Users`
- 4-Boxen-Grid (`grid-cols-2 sm:grid-cols-4`) unter der Lede:
  - Icon in 40×40 rounded-xl Container mit accent-tinted Background
  - Label in font-mono mit textShadow
  - Canonical Card-Hover: `hover:scale-[1.015] hover:shadow-[0_0_20px_var(--accent-glow)] hover:border-[var(--border-accent)]`
  - `backdrop-filter: blur(8px)` für Card-Lesbarkeit über LabeledNodes-BG
- `handleTypeClick` ruft `onTypChange?.(slug)` und scrollt zu `[data-section="partner-tab-system"]`

### `partner-client.tsx`
- `<PartnerHeroSection onTypChange={handleTypChange} />` — vorherige Tab-State-Plumbing wiederverwendet

## Decisions

- Icon-Mapping: Building2 (Unternehmen) / Landmark (Stiftungen) / GraduationCap (Hochschulen) / Users (Initiativen) — Standard-Lucide-Semantik
- Boxen sind `<button>`, nicht `<Link>` — wir bleiben auf `/partner` und triggern den existierenden Tab-State (deep-link via `?typ=` würde Page-Reload erzwingen)
- Hero-min-height (`calc(100vh-5rem)`) bleibt — Boxen vergrößern den Inhalt aber Box ist zentriert, oberhalb der Fold visuell sicher

## Must-Haves

- [x] 4 Boxen rendern mit Lucide-Icons in CI-Akzent
- [x] Click → Tab-State updates + smooth-scroll zum Tab-System
- [x] Canonical Card-Hover (kein inline JS)
- [x] Hero-Pattern aus AGENTS.md eingehalten (LabeledNodes + max-w-4xl + --fs-display)
- [x] Keyboard-fokussierbar (`focus-visible` outline)
- [x] aria-label pro Box

## Out-of-Scope

- Tab-Inhalte selbst (steht in `partner-tab-content.tsx`, unverändert)
- Icon-Animations on hover (kein Call-Feedback dazu)
