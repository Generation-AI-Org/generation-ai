# Plan 22.8-02 — Token-Source + Component-Compliance-Fix

> **Status: depends-on-audit.** Konkreter Scope wird gefüllt nach Plan 22.8-01 durchgelaufen + Luca-Review der Findings. Skelett bleibt bis dahin minimal.

---

## Objective (TBD nach Audit)

Auf Findings aus `22.8-01-AUDIT.md` reagieren:
1. Components die Tokens nicht nutzen → an Component-Source umstellen auf DS-Tokens
2. Components die abweichende Hover-States haben → auf canonical Pattern (`scale(1.015) + Shadow + Border-Fade`) umstellen
3. Hardcoded Farben → durch CSS-Variablen-Referenzen ersetzen
4. Logo-Hover-States vereinheitlichen (rot/accent bei Hover, dauerhaft sichtbar)

**Wichtig: Fix passiert in der canonical Component-Source ODER im Token, nicht ad-hoc auf Konsumenten-Seiten.** Eine Änderung in `packages/ui/Card.tsx` propagiert automatisch zu allen Pages die Card nutzen — das ist gewollt.
