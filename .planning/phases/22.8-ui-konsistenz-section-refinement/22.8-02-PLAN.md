# Plan 22.8-02 — Token-Source + Component-Compliance-Fix

> **Status: ready, autonomous.** Concrete scope from `22.8-01-AUDIT.md` (24 findings, all approved by Luca 2026-04-25). Foundation-First: every fix lands at component-source / token level, never ad-hoc per page.

---

## Objective

Repair the 5 DS-drift clusters identified by Plan 22.8-01:

1. **Status-Color tokens** — replace hardcoded Tailwind `red-500/green-500/emerald-500` with `--status-error` / `--status-success` (light-mode-blocking in tools-app auth/settings).
2. **Card-Hover unification** — collapse 3 competing hover languages (`scale-[1.015]`, `-translate-y-[2/3px]`, `-translate-y-0.5`) onto the canonical `hover:scale-[1.015] + accent-glow + border-accent`.
3. **Logo-Hover spec** — add additive `interactive` prop to canonical `Logo` component; opt-in at header (replaces ad-hoc class).
4. **Border-Radius standardization** — 4 primary CTAs from `rounded-md`/`rounded-lg` → `rounded-full`.
5. **Inline-JS hover removal** — replace `onMouseEnter/Leave/Down/Up` with Tailwind `hover:` / `active:` classes in 2 form submit buttons.

Plus low-risk token-cleanup nice findings (A-N01..A-N05) that are 1-line fixes.

**Foundation-First rule (CONTEXT.md):** Fix at canonical source. A change in `packages/ui/Logo.tsx` automatically propagates to all consumers — that is the goal, not a side-effect.
