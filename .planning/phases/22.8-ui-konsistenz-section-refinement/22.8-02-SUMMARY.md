## Other checkpoint items / spot-check uncertainty

- The new card-hover pattern uses `scale-[1.015]` which is a **subtler** lift than the previous `-translate-y-[3px]`. On `offering-section.tsx` and `problem-block-section.tsx` (the two "drift" cards), the previous behavior had `0_0_32px` glow vs the canonical `0_0_20px` — the cards may now feel slightly less prominent. Per audit decision A-S06/A-S07 this is intentional alignment, not a regression. If Luca prefers the bolder feel, the path forward is to add a documented `feature-card` DS variant with the bigger glow — out of 22.8 scope.
- The `community-preview-section.tsx` cards changed `motion-reduce:hover:translate-y-0` → `motion-reduce:hover:scale-100` (semantically equivalent for the new transform property). Reduced-motion users still get no animation.
- `auth/error/page.tsx` button changed from `rounded-md` to `rounded-full` AND keeps shadcn `bg-primary` (alias to `--accent` via globals.css line 18). Visually correct in both themes.
- `tools-app/error.tsx` + `not-found.tsx` use `bg-neon` directly — kept as-is since these brand classes are dark-mode-bg-only by tools-app convention. Only the radius changed.
