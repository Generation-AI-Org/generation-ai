## Decisions Made

### D-22.8-03-A: schema.ts logo-URL auf dynamic /opengraph-image statt brand/logos/og-image-default.png

Pre-existing JSON-LD-Refs in `apps/website/lib/schema.ts` zeigten auf `https://generation-ai.org/og-image.jpg` (Plan-Scope: zu löschen). Statt sie auf das neue static `/og-image-default.png` zu setzen (was die Website nicht hostet — nur tools-app), zeige ich auf `/opengraph-image` (Phase 22.6 Track B dynamic OG-route). Vorteile: (1) canonical brand-source, (2) kein neuer static-Pfad, (3) Logo + Brand-Tagline werden zusammen ausgeliefert. Gilt für `Organization.logo` + `Article.publisher.logo`.

### D-22.8-03-B: favicon.ico war pre-existing byte-identisch in beiden Apps

Beide Apps hatten `app/favicon.ico` schon vor diesem Plan vorliegen, und beide waren byte-identisch zum neuen `brand/logos/favicons/favicon.ico`. `cp` schreibt das File, aber Git registriert keinen Diff. SUMMARY dokumentiert das explizit damit klar ist, dass die Convention-Files alle vorhanden sind.
