## Color

Source: `brand/DESIGN.md §C`, `brand/tokens.json` — fully pre-populated.

### 60 / 30 / 10 Split

| Role | Light value | Dark value | Usage |
|------|------------|------------|-------|
| Dominant (60%) | `#F6F6F6` | `#141414` | Body background, page canvas |
| Secondary (30%) | `#FFFFFF` / `#EFEFEF` | `#1C1C1C` / `#222222` | Cards, modals, elevated surfaces, nav |
| Accent (10%) | `#F5133B` | `#CEFF32` | Primary buttons, active states, focus indicators, header CTA links |

Accent is reserved for: primary action buttons, selected/active nav items, focus ring on interactive elements, inline links, header decorative band, icon emphasis on key interactions.

Accent is NOT used on: body backgrounds, card borders, page-level dividers, muted text, disabled states.

### Semantic Status Colors

| Role | Hex | Usage |
|------|-----|-------|
| Error | `#DC2626` | Form validation errors, destructive action confirmations, error toasts |
| Success | `#16A34A` | Confirmation toasts, success states |
| Warning | `#F59E0B` | Non-blocking warnings |
| Info | `#2563EB` | Informational callouts |

These are separate from brand colors and must never be substituted with brand accent colors.

### Dark Mode Strategy

- Default: system `prefers-color-scheme` — dark is default (`:root` in `base.css`)
- Light mode: `.light` class on `<html>` (already implemented)
- User toggle overrides via `localStorage`, persisted across sessions
- No background-flood: Header band uses `--bg-header` (Rosa/Blau), body always uses `--bg`
- Shadow opacity in dark mode: multiply default shadow alpha by 4–5× (`rgba(0,0,0,0.4–0.5)`)

### Interaction States

| State | Rule |
|-------|------|
| Hover | `--accent-hover` for primary; lightness −8% or alpha overlay for neutral elements |
| Active | Scale `0.98` + lightness −16% |
| Focus | `outline: 2px solid var(--text)` + `outline-offset: 2px` (neutral-high-contrast, not brand-colored) |
| Disabled | `opacity: 0.5` + `cursor: not-allowed` |

**Note:** `globals.css` in website app already applies `outline: 2px solid var(--accent)` on `:focus-visible`. This must be updated to `var(--text)` during migration to match the brand contract (fix was applied in workshop session per CONTEXT.md — verify it persists).

## Logo Component Contract

Source: `brand/DESIGN.md §F` — fully pre-populated.

### Component API

```tsx
<Logo variant="wide" colorway="auto" context="header" />
<Logo variant="wide" colorway="red" height={40} />
<Logo variant="wide" colorway="neon" size="md" />
```

| Prop | Type | Values | Default |
|------|------|--------|---------|
| `variant` | string | `"wide"` | `"wide"` |
| `colorway` | string | `"auto"` \| `"red"` \| `"neon"` \| `"black"` \| `"white"` \| `"pink"` \| `"blue"` \| `"pink-red"` \| `"red-on-pink"` \| `"blue-neon"` \| `"neon-on-blue"` \| `"sand"` | `"auto"` |
| `context` | string | `"header"` \| `"footer"` \| `"mail"` | `"header"` |
| `size` | string | `"sm"` (32px) \| `"md"` (40px) \| `"lg"` (56px) | `"md"` |
| `height` | number | explicit px override | — |
| `className` | string | passthrough | — |

`colorway="auto"` resolution matrix (executor must implement):

| Context prop | Theme | Resolved colorway |
|-------------|-------|------------------|
| `header` | light | `red` |
| `header` | dark | `neon` |
| `footer` | light | `black` |
| `footer` | dark | `white` |
| `mail` | light | `red` |
| `mail` | dark | `neon` |

### Logo Usage Rules (enforce in component)

- Minimum height: 32px digital. Throw console.warn if height < 32.
- Never apply CSS transforms: `scale`, `rotate`, `skew`, `filter: drop-shadow`, `filter: blur`
- Clearspace: component must render with `margin: 0` — caller is responsible for ½-logo-height clearspace
- Never recolor SVG inline (no CSS `fill` overrides) — use named colorway variants only
- Render as `<img>` (not inline SVG) to prevent accidental CSS style leakage into SVG paths

## Migration Guidance

This section is for executor reference — not a new screen contract but a migration checklist.

### Font Migration (both apps)

1. Remove Inter import from `apps/*/app/layout.tsx`
2. Add `Geist` + `Geist Mono` via `next/font/google`
3. Pass font CSS variables as `className` on `<html>` element
4. Update `globals.css @theme inline`: `--font-sans: var(--font-geist-sans), ...`
5. Update `base.css`: `--font-family-mono` fallback to Geist Mono (remove Cascadia/Fira Code references)
6. **CSP safety:** `next/font` self-hosts — no external font requests, no CSP change needed

### Radix Colors Integration

1. Install `@radix-ui/colors` in `packages/config` (or workspace root)
2. Import light + dark palettes in `base.css`:
   ```css
   @import "@radix-ui/colors/slate.css";
   @import "@radix-ui/colors/slate-dark.css";
   @import "@radix-ui/colors/slateA.css";
   ```
3. Map Radix CSS variables to the semantic layer in `base.css`
4. Remove any hardcoded neutral hex values in component files — replace with `var(--slate-N)`

### Microcopy Migration Pass

Audit both apps for strings that conflict with `brand/VOICE.md`. Key replacements:

| Find (old pattern) | Replace with (VOICE.md canonical) |
|-------------------|----------------------------------|
| "Ein Fehler ist aufgetreten" | "Ups, da ist was schiefgelaufen. Probier's nochmal!" |
| "Erfolgreich gespeichert" | "Passt, gespeichert!" |
| "Einen Augenblick bitte" | "Einen Moment…" |
| "Sind Sie sicher?" | "Sicher? Das lässt sich nicht rückgängig machen." |
| "Hier sind noch keine Einträge" | "Hier ist noch nichts. Leg los mit deinem ersten Tool." |
| Buttons with "!" suffix | Remove exclamation mark |

## Copywriting Contract

Source: `brand/VOICE.md` — fully pre-populated. No new screens in this phase. Contract documents the canonical strings the microcopy-pass must enforce.

| Element | Copy |
|---------|------|
| Primary CTA (generic action) | Verb-first, no exclamation mark — e.g. "Loslegen", "Anmelden", "Speichern" |
| Empty state heading | "Hier ist noch nichts." |
| Empty state body | "Leg los mit deinem ersten Tool." |
| Error state (server) | "Ups, da ist was schiefgelaufen. Probier's nochmal!" |
| Error state (user input) | "Hmm, [spezifisches Problem]." e.g. "Hmm, die Mail-Adresse passt noch nicht ganz." |
| Success (saved) | "Passt, gespeichert!" |
| Loading state | "Einen Moment…" |
| Destructive confirmation | "Sicher? Das lässt sich nicht rückgängig machen." |
| Destructive button label | "Löschen" (never "Unwiderruflich entfernen") |
| Session expired | "Deine Session ist abgelaufen. Log dich nochmal kurz ein." |

Voice rules enforced in this phase:
- No exclamation marks in button labels
- No emojis in UI (only in welcome mail where documented)
- Real Umlauts: ö/ä/ü/ß — never oe/ae/ue/ss
- No "leider", "erfolgreich", "möchten Sie"
- Brand interjections: "Ups" (server errors), "Hmm" (user input errors), "Passt" (success), "Let's go" (onboarding moments) — max 1 per message
