| sm | 8px (`--space-2`) | Compact element spacing, badge padding |
| md | 16px (`--space-4`) | Default element spacing, card padding unit |
| lg | 24px (`--space-6`) | Card padding (px-6 py-6), section inner padding |

## Tab System — Interaction Contract

### ARIA Pattern
- Container: `role="tablist"` with `aria-label="Partnertypen"`
- Each tab: `role="tab"` + `aria-selected={true|false}` + `aria-controls="{id}-panel"`
- Each panel: `role="tabpanel"` + `id="{typ}-panel"` + `aria-labelledby="{typ}-tab"` + `tabIndex={0}`
- Hidden panels: `hidden` attribute (not `display:none` via CSS class — native HTML attribute ensures accessibility)

### Tab Labels (exact)
1. **Unternehmen** (default, `?typ=unternehmen`)
2. **Stiftungen** (`?typ=stiftungen`)
3. **Hochschulen** (`?typ=hochschulen`)
4. **Initiativen** (`?typ=initiativen`)

### URL Sync
- Tab click: `window.history.pushState(null, "", "?typ={slug}")` — no full navigation, no scroll reset
- On mount: read `useSearchParams()` to set initial active tab; default to `unternehmen` if param absent or invalid
- SSR: page.tsx passes initial `searchParams.typ` to client wrapper for hydration parity

### Tab Visual States
- **Inactive tab:** `text-text-muted`, no border, background transparent, `font-mono font-bold text-[14px] tracking-[0.02em]`
- **Active tab:** `text-text`, 2px bottom border `border-b-2 border-[var(--accent)]`
- **Hover (inactive):** `text-text-secondary`, transition `var(--dur-fast)` `var(--ease-out)`
- **Focus-visible:** `outline: 2px solid var(--text); outline-offset: 2px; border-radius: 4px` (DS global)
- Keyboard: Arrow Left/Right navigates between tabs, Enter/Space activates

### Tab Content Transition
- On tab switch: fade-out active panel (opacity 0, `duration: var(--dur-fast)` = 150ms), then fade-in new panel (opacity 1, y from 8px, `duration: var(--dur-normal)` = 300ms, ease `var(--ease-out)`)
- No scroll reset on switch — `window.scrollY` preserved
- `useReducedMotion`: skip fade, swap instantly (no animation)
- Implementation: CSS transition + `key` prop change triggers React remount (simple), OR Framer Motion `AnimatePresence` mode="wait"

### Mobile Tab Layout
- **Horizontal scroll** (chosen pattern — consistent with tool-showcase marquee) — not dropdown
- `overflow-x-auto scrollbar-hide` on tab rail container
- Tab rail: `flex flex-nowrap gap-1 px-6` — tabs do not wrap to second line
- Minimum tab touch target: 44px height (`min-h-[44px]` per DS touch target rule)
- Active tab scrolls into view on mount (if deep-linked): `element.scrollIntoView({ inline: "center", behavior: "smooth" })`

## Contact Form — Interaction Contract

### Fields (in order)
1. **Name** — text input, required, label "Name"
2. **E-Mail** — email input, required, label "E-Mail"
3. **Organisation** — text input, required, label "Organisation"
4. **Ich interessiere mich als …** — `<select>`, required. Options: Unternehmen / Stiftung / Hochschule / Initiative. Pre-selected to match active tab on mount; user-changeable. label "Ich interessiere mich als …"
5. **Nachricht** — `<textarea>`, optional, label "Nachricht (optional)", rows 4
6. **Honeypot** — `<input type="text" name="website" tabIndex={-1} aria-hidden="true" className="sr-only">` — hidden from humans, if non-empty → silent reject on server action

### Validation
- Client-side: required fields show error inline on blur + on submit attempt
- Error message style: `text-[var(--status-error)] text-sm` below the input, `role="alert"` or `aria-live="polite"`
- Email: pattern `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- Submit button disabled while submitting (loading state)

### States
- **Idle:** form visible, submit button "Anfrage senden" enabled
- **Submitting:** button shows spinner or "Wird gesendet…", all inputs `disabled`
- **Success:** form replaced by success card (not inline success message — full replacement)
- **Error (server):** inline error banner above submit button, form remains editable

### Form Card
- Container: `rounded-2xl border border-border bg-bg-card px-6 py-8 sm:p-10`
- Submit button: Primary pill style — `bg-[var(--accent)] text-[var(--text-on-accent)] rounded-full px-6 py-2.5 font-mono font-bold text-[14px] tracking-[0.02em]` + `hover:shadow-[0_0_20px_var(--accent-glow)] hover:scale-[1.03] active:scale-[0.98] transition-all duration-300`

## Motion Contract

All motion uses DS tokens:
- `--dur-fast`: 150ms — tab active indicator transition, hover color changes
- `--dur-normal`: 300ms — tab content fade-in, form field focus transitions
- `--dur-slow`: 500ms — not used in this phase
- `--ease-out`: `cubic-bezier(0.16, 1, 0.3, 1)` — viewport entry animations (whileInView)
- `--ease-in-out`: `cubic-bezier(0.65, 0, 0.35, 1)` — tab content swap

Entry animations (viewport `whileInView`, `once: true`, `margin: "-10% 0px"`): `opacity: 0 → 1, y: 16 → 0, duration: 0.6s, ease: var(--ease-out)` — identical to TrustSection and AboutKontaktSection patterns.

Hero entry: `initial={opacity: 0, y: 16}`, `animate={opacity: 1, y: 0}`, `duration: 0.7, ease: "easeOut"` — matches AboutHeroSection.

`useReducedMotion()` gate: all animations collapse to `{}` (no transform, instant opacity). Tab content swap becomes instant. Marquee pause via `animation-play-state: paused` in CSS (already established in globals.css).

## Accessibility Contract

- ARIA-Tabs pattern: `role="tablist"`, `role="tab"` + `aria-selected`, `role="tabpanel"` + `aria-labelledby` (full spec in Tab System section above)
- Form: all inputs have associated `<label>` (explicit `htmlFor` / `id` pairs)
- Honeypot: `tabIndex={-1}` + `aria-hidden="true"` — never focused by keyboard or announced by SR
- Contact links on person cards: `aria-label="E-Mail an {name}"` / `aria-label="LinkedIn-Profil von {name}"` (since icon-only variant would be ambiguous)
- Skip link: inherited from Header (already in layout)
- Focus management: on tab switch, focus moves to the newly revealed tabpanel (via `tabPanel.focus()` after content swap)
- Error announcements: `role="alert"` on error messages so SR reads immediately without explicit focus
- `data-section` attributes on all `<section>` elements for Playwright selectors
