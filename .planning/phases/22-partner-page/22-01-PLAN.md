# Plan 22-01 — React Email Templates

## Objective

Create two React Email templates in `@genai/emails` package:
1. `PartnerInquiryEmail` — admin notification sent to `admin@generation-ai.org`
2. `PartnerInquiryConfirmationEmail` — confirmation sent to the person who submitted the form

Export both from `packages/emails/src/index.ts`.

## Context

- D-09: React Email Template in `@genai/emails` Package for admin notification
- D-10: Confirmation mail to partner-lead from `noreply@generation-ai.org`
- Template pattern: `packages/emails/src/templates/confirm-signup.tsx`
- Shared components: `Layout`, `EmailButton`, `tokens`, `fontStack` from `packages/emails/src/index.ts`

## Threat Model

- No user-controlled content rendered as raw HTML — React Email components escape all props
- Template props are typed (TypeScript) — runtime validation happens in the Server Action (Plan 22-02)
- No sensitive data (passwords, tokens) included in email content
