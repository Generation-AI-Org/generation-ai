<task type="checkpoint:human-action" gate="blocking">
  <action>Better Stack Account und Monitors einrichten</action>
  <instructions>
1. Gehe zu https://betterstack.com/uptime und erstelle einen Free Account

2. Lege 4 HTTP Monitors an (Monitors -> Add Monitor -> HTTP):

   | Name | URL | Interval |
   |------|-----|----------|
   | Website | https://generation-ai.org | 3 min |
   | Tools App | https://tools.generation-ai.org | 3 min |
   | Community | https://community.generation-ai.org | 3 min |
   | API Health | https://tools.generation-ai.org/api/health | 3 min |

3. Alerting konfigurieren:
   - E-Mail-Alert bei Downtime aktivieren (Default)
   - Recovery-Alert aktivieren ("Site is back up")

4. Optional: Status Page erstellen (Settings -> Status Page)
   - Subdomain waehlen (z.B. status.generation-ai.org)
  </instructions>
  <resume-signal>Sage "Better Stack Monitors aktiv" wenn alle 4 Monitors laufen</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Sentry Alert-Regeln konfigurieren</action>
  <instructions>
Fuer beide Sentry-Projekte (website + tools-app):

1. Gehe zu Project -> Alerts -> Create Alert Rule

2. **Issue Alert: Neuer Fehler** (pro Projekt)
   - When: "A new issue is created"
   - Then: "Send a notification via Email"
   - Name: "New Issue Alert"

3. **Issue Alert: Error Spike** (pro Projekt)
   - When: "The issue is seen more than 100 times in 1 hour"
   - Then: "Send a notification via Email"
   - Name: "Error Spike Alert"

4. Notification Settings pruefen:
   - Settings -> Notifications -> E-Mail aktiviert
  </instructions>
  <resume-signal>Sage "Sentry Alerts konfiguriert" wenn die Alert-Regeln aktiv sind</resume-signal>
</task>
