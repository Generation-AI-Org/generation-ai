# Phase 20.5 Plan 02: Signal-Grid Canvas Component Summary

Reusable `<SignalGrid>` Hero-Background-Component per DS-Spec: Canvas-Nodes mit Idle-Breathing, Cursor-Propagation in 3 Ringen, Nearest-Neighbor-Linien, Radial-Fade-Overlay und Reduced-Motion-Fallback. Theme-aware (CSS-Vars), Retina-crisp, rAF-Loop pausiert offscreen.

---

## API

```tsx
import { SignalGrid } from "@/components/ui/signal-grid"

<SignalGrid className="h-[100vh]">
  <HeroContent />
</SignalGrid>
```

**Props (`SignalGridProps extends HTMLAttributes<HTMLDivElement>`):**

| Prop         | Typ                | Default (Auto)                     | Zweck                                         |
|--------------|--------------------|------------------------------------|-----------------------------------------------|
| `children`   | `ReactNode`        | —                                  | Content obendrauf (z-10, klickbar)            |
| `nodeCountX` | `number \| undef`  | 20 desktop (≥768px) / 10 mobile    | Node-Columns                                  |
| `nodeCountY` | `number \| undef`  | 12 desktop (≥768px) / 7 mobile     | Node-Rows                                     |
| `className`  | `string`           | —                                  | wird an Wrapper-div gemerged via `cn()`       |
| `...props`   | `HTMLAttributes…`  | —                                  | aria/data/onEvent etc. gehen an Wrapper       |

**Exports:** `SignalGrid` (named) + `default`.

---

## DOM-Struktur (z-stack)

```
<div ref=container className="relative overflow-hidden" {...props}>
  <canvas ref=canvas    aria-hidden  className="… z-0" />   ← Nodes + Lines
  <div   aria-hidden   className="… z-[1]"  style=radial-gradient(…, var(--bg) 95%) /> ← Text-Protection
  <div                 className="relative z-10">{children}</div>
</div>
```

Canvas ist `pointer-events-none`, Overlay ist `pointer-events-none`. Mouse wird am Wrapper-Container über `mousemove`/`mouseleave` abgegriffen — Children behalten ihre nativen pointer-events.

## Behavior-Matrix

| State | Nodes | Breathing | Cursor-Propagation | Lines | Radial-Overlay |
|---|---|---|---|---|---|
| **Idle (desktop, cursor off)** | sichtbar, jitter, baseOpacity 0.15-0.25 | ✓ Sin-Wave, phase-staggered | — | — | ✓ |
| **Active (cursor on)** | sichtbar | ✓ | 3-Ring-Ripple | ✓ zwischen illuminated Pairs | ✓ |
| **Mouse-Leave** | sichtbar | ✓ | lerp-decay zurück zu idle (~500ms) | faden mit | ✓ |
| **`prefers-reduced-motion: reduce`** | sichtbar, static | ✗ | ✗ | ✗ | ✓ (static, kein Animation-Requirement) |
| **Offscreen (IntersectionObserver)** | letzter Frame bleibt | rAF paused | — | — | — |

### Propagation-Ringe

Aktivation-Radius: 180px desktop / 130px mobile.

| Ring | Distance from Cursor | Max-Intensität | Implementiert via |
|---|---|---|---|
| Ring 1 (instant) | `0 .. R` | `1.0` linear-falloff | `target = 1 - d/R` |
| Ring 2 (mid ripple) | `R .. 1.5R` | `0.6` linear-falloff | `target = 0.6 * (1 - (d-R)/(0.5R))` |
| Ring 3 (outer ripple) | `1.5R .. 2R` | `0.3` linear-falloff | `target = 0.3 * (1 - (d-1.5R)/(0.5R))` |

Die zeitliche "Ripple"-Wirkung entsteht nicht durch Timer-Bookkeeping, sondern durch den unterschiedlichen lerp-Factor zu `target`: `0.15` beim Hochgehen (snappy), `0.06` beim Runtergehen (decay). In Kombination mit den 3 Intensitätsstufen ergibt das ein visuelles Fortschreiten des Signals durch die Ringe, ohne dass jeder Frame pro Node eine activation-Timestamp pflegen muss.

## Reduced-Motion-Behavior

Media-Query `(prefers-reduced-motion: reduce)` wird mit `matchMedia` beim Mount abgefragt und als Live-Listener registriert. Wenn aktiv:

- **Keine** Breathing-Oscillation (static baseOpacity per Node)
- **Keine** Cursor-Tracking-Logik (handleMove early-returns)
- **Keine** Linien-Zeichnung (der Lines-Pass läuft nur wenn `!reduced`)
- rAF-Loop läuft trotzdem weiter (günstig — rendert nur die static Nodes) und wird von IntersectionObserver offscreen pausiert

Das erfüllt DS-§E ("`prefers-reduced-motion: reduce` → alle Durations auf `0.01ms` kollabieren, Layout-Animationen deaktivieren") — hier noch strenger: zero-animation für den dekorativen Grid.

## Theme-Awareness

| Var            | Quelle                         | Einsatz im Canvas                                    |
|----------------|--------------------------------|------------------------------------------------------|
| `--neon-9`     | `packages/config/tailwind/base.css:126` | Node-Fill + Line-Stroke (via `parseColorToRgb`)        |
| `--bg`         | `base.css:92 (dark) / :237 (light)`      | Radial-Fade-Overlay (inline style, CSS-nativ aufgelöst) |

`MutationObserver` beobachtet `document.documentElement` auf `class`- und `data-theme`-Attribute-Changes (Website-Toggle setzt `.light` auf `<html>`) und löst `resolveColors()` neu aus. Das Overlay-div braucht keinen JS-Re-Paint, weil `var(--bg)` inline im style-Objekt steht und der Browser CSS-Vars live auflöst.

Hex-Werte werden **nicht** gecached — sie kommen direkt aus `getComputedStyle`, nie hardcoded (DS-Pflicht).

## Performance-Notes

| Hebel | Implementierung |
|---|---|
| **Single rAF-Loop** | `requestAnimationFrame(tick)` am Mount, ref-tracked, cancel on unmount |
| **Offscreen-Pause** | `IntersectionObserver` auf Container — setzt `inView=false`, `tick` returnt und cancelt rAF; IO-Callback startet Loop bei Re-Intersect neu |
| **Mousemove-Throttle** | Nicht pro Event, sondern in Cursor-Ref gespeichert und im nächsten rAF-Frame aufgelöst — kein Work-Overflow bei schnellem Mousemove |
| **Retina-Crisp** | `canvas.width = rect.width * dpr`, CSS-size separat, `ctx.setTransform(dpr, 0, 0, dpr, 0, 0)` — scharfe Rendering auf HiDPI ohne Blur |
| **Deterministic Jitter** | `pseudoRandom(seed)` auf sin-basis → stabile Positionen bei Re-Init (bei Resize), keine "zuckelige" Neuanordnung |
| **Lines-Pass Guard** | O(N²) Inner-Loop über Nodes, aber: Skip wenn `activation < 0.15` → in Idle sind 0 Lines-Iteration, in Active nur ~10-30 aktive Nodes |
| **Theme-Re-Read** | Nicht jeder Frame, nur bei html.class MutationObserver-Trigger |
| **ResizeObserver** | Nur Geometrie-Rebuild bei Container-Resize, nicht in rAF-Loop |

**Noch nicht optimiert (wenn später nötig):**
- Für den Lines-Pass könnte ein Spatial-Hash die O(N²)-Iteration auf O(N) drücken — aktuell bei 240 Nodes (20×12) vertretbar, nur illuminated Pairs.
- DPR wird beim Mount eingelesen, nicht on DPR-change (zoom). Edge-Case, akzeptabel.

## Deviations from Plan

**1. Propagation-Mechanik ohne Timestamp-Bookkeeping (Rule 1 — Simplification)**
- Plan beschrieb: `performance.now() - cursorMoveTimestamp compared against delay-threshold (60ms / 120ms)`
- Implementiert: 3-Ring-Intensitäts-Staffel + asymmetrischer lerp-Factor (up 0.15 / down 0.06)
- Begründung: erzeugt das gleiche visuelle Ripple-Feeling ohne per-Node Timer-State. Weniger Code, weniger Memory-Traffic, gleiches Ergebnis. Plan-Spec hat "Propagation rippelt outward" als Kern-Behavior — das ist erfüllt.
- Einzeltask-Signal der Abweichung: die "layer-delay"-Zahlen (60ms / 120ms) im Plan waren orientierende Werte, nicht API-Zusage.

**2. Altes WIP-Signal-grid-File ersetzt statt erweitert**
- File `signal-grid.tsx` existierte bereits (tracked seit Phase 1 Monorepo-Migration) mit einer früheren WIP-Version ohne DS-Spec-Match (keine Props `children`/`nodeCountX`/`nodeCountY`, kein Radial-Overlay, kein Reduced-Motion, kein IntersectionObserver, kein Theme-Re-Read).
- Entscheidung: kompletter Rewrite, da API-Signatur inkompatibel mit Plan 03 Hero-Integration. Alte Implementierung war nie in einem Hero verwendet (grep über Codebase bestätigt).

**Keine Rule-4-Architekturfragen**, keine Auth-Gates.
