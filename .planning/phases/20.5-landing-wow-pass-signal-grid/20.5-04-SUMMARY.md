# Phase 20.5 Plan 04: Discrepancy Polish Pass Summary

Polish-Pass des Sticky-Scroll Split-Line-Charts aus Phase 20-06: smootherer useSpring, honest Paar-IDs statt strapazierter Kategorie-Labels, HTML-Legend + Pair-Caption unter dem Chart für Mobile-Readability. Reduced-Motion-Pfad unverändert, alle Farben weiterhin über CSS-Vars.

## Was wurde geändert

Eine Datei, ein atomic commit.

### `apps/website/components/sections/discrepancy-section.tsx`

**1. Spring-Tuning (Task 1)**

| Parameter | Vorher | Nachher | Rationale |
|---|---|---|---|
| stiffness | 120 | 140 | Responsiver — Chart reagiert schneller auf Scroll-Input |
| damping | 30 | 32 | Leicht mehr Dämpfung gegen Wobble |
| mass | 0.5 | 0.4 | Leichteres Gefühl, kein Träge-Effekt |

Kein `restDelta` / `restSpeed` gesetzt — Defaults reichen. Die Spring wraps weiterhin `scrollYProgress` direkt (wie vor Plan-04), die `useTransform`-Outputs erben die geglättete Progress-Kurve.

**2. Phasen-Range-Overlaps (Task 1)**

Alle Phasen-Überlappungen auf ≥ 0.05 geweitet — keine harten Step-Grenzen zwischen aufeinanderfolgenden Effekten.

| Phase | Vorher (start-end) | Nachher (start-end) |
|---|---|---|
| title fade | 0.00–0.22 | 0.00–0.22 (unverändert) |
| upper line draw | 0.15–0.50 | 0.15–0.50 (unverändert) |
| upper dots fade | 0.32–0.52 | 0.30–0.52 (Overlap mit line draw 0.2 vorher 0.18) |
| lower line draw | 0.42–0.75 | 0.40–0.75 |
| lower dots fade | 0.58–0.78 | 0.55–0.78 |
| area fade | 0.68–0.90 | 0.65–0.90 |
| "DIE LÜCKE" label | 0.72–0.90 | 0.70–0.92 |
| closer fade/y | 0.82–0.98 | 0.80–0.98 |

Reduced-Motion-Path: alle `useTransform`-Input-Arrays bleiben strukturell gleich (2 oder 3 stops), Output-Array konstant auf Final-Wert bei `prefersReducedMotion === true` → kein Unterschied durch die range-shifts.

**3. Honest X-Axis (Task 2)**

Option A aus dem PLAN gewählt: neutrale Paar-IDs "01 / 02 / 03" (Geist Mono, tracked-out) statt der strapazierten Wort-Labels "Nachfrage / Vergütung / Ausbildung". Die Paar-Semantik lebt jetzt in einer HTML-Caption unter dem Chart.

X-Axis-Title: "DIMENSION" → "PAAR".

Rationale: Die Daten-Paare sind in Wahrheit
- (01) KI-Talent-Nachfrage (7×) vs. Anfänger-Level (83,5 %)
- (02) Lohnaufschlag (56 %) vs. „Studium bereitet nicht vor" (75 %)
- (03) ungenutztes Potenzial (73 %) vs. intensive KI-Lehre (6,4 %)

Es gibt keine saubere 1-Wort-Kategorie, die beide Seiten eines Paars fair beschreibt — "Vergütung" passt nur auf den Wirtschaftswert, nicht auf die Studi-Seite. Paar-IDs sind rein strukturell, die narrative Arbeit macht die Caption.

**4. Honest Y-Axis (Task 2)**

Y-Tick-Labels (hoch / mittel / niedrig) + Y-Title "LEVEL" bleiben unverändert. Zusätzlich in der Caption expliziter Hinweis:

> Y-Achse symbolisch (niedrig / mittel / hoch) — die Werte mischen × und %.

Das verhindert, dass ein Leser die Y-Positionen als % liest (wo 7× nicht sinnvoll wäre).

**5. Legend + Caption als HTML unter dem Chart (Task 2 + 3)**

Die SVG-interne Legend (`<g transform="translate(58, -2.5)">`) ist weg. Neu: HTML-Block direkt nach dem `</svg>`:

- **Legend**: `flex flex-wrap items-center justify-center gap-x-5 gap-y-2` — bricht auf Mobile von selbst auf 2 Zeilen, stackt nicht über Chart-Linien.
  - Label-Texte: "Markt-Signal — Was die Wirtschaft will" / "Studi-Realität — Was Studis mitbringen" (konsistenter Narrative-Frame, kein "Wirtschaft/Studis" allein).
- **Pair-Caption**: `max-w-2xl` font-mono, Paar-Zuordnung nummeriert + Y-Scale-Note.
- `role="list"` / `role="listitem"` für A11y.

**6. Mobile-Readability (Task 3)**

| Element | Desktop (≥ 640px) | Mobile (< 640px) |
|---|---|---|
| X-Tick-Labels "01/02/03" | sichtbar (`max-sm:hidden`) | hidden — zu klein zum Lesen bei voller Chart-Breite |
| X-Title "PAAR" | sichtbar | hidden |
| Y-Tick-Labels (hoch/mittel/niedrig) | sichtbar | sichtbar (klein aber noch lesbar bei fontSize 2.2px in viewBox-units) |
| Legend | unter Chart, 1 Zeile | unter Chart, flex-wrap → 2 Zeilen |
| Pair-Caption | `text-xs`, 1 Zeile + Block-Note | `text-[11px]`, 2 Zeilen (3 Paare wrap'en) + extra mt-2 für Note |
| Chart-Height (Scroll-Distance) | `h-[200vh]` unverändert | `h-[150vh]` unverändert |

**7. CSS-Vars / Theme-Awareness (Task 2 Verification)**

- Axes + Ticks: `var(--text-muted)`

Keine Transforms über Scroll. Bestätigt durch Code-Review (keine Änderung am Branching).
