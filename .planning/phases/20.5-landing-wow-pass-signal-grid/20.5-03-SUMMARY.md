# Phase 20.5 Plan 03: Hero SignalGrid + DS-aligned Button States Summary

Hero-Section verwendet jetzt `<SignalGrid>` statt `<GridBackground>`, und die Primary-CTA ist komplett DS-konform (Pill-Radius + Scale-Hover + Scale-Press + --accent-glow Shadow + --ease-out Timing). Alte `grid-background.tsx` bleibt erhalten mit @deprecated-Header für mögliche spätere Verwendung.

---

## Vor / Nach — Hero-Section

### Background-Swap

**Vorher:**
```tsx
import { GridBackground } from "@/components/ui/grid-background"
…
<GridBackground className="min-h-[80vh]">
  <motion.div …>{heroContent}</motion.div>
</GridBackground>
```

**Nachher:**
```tsx
import { SignalGrid } from "@/components/ui/signal-grid"
…
<SignalGrid className="flex min-h-[80vh] flex-col items-center justify-center">
  <motion.div …>{heroContent}</motion.div>
</SignalGrid>
```

**Warum Wrapper-Flex-Classes:** `GridBackground` hatte `flex h-[100vh] flex-col items-center justify-center` hardcoded in der Component. `SignalGrid` ist bewusst layout-agnostisch (nur `relative overflow-hidden`). Damit die zentrierte motion.div vertikal gemittet im 80vh-Container sitzt, wandert das Flex-Layout an den Wrapper. API-saubererer Split.

### Button-States (Primary-CTA)

**Vorher:**
```tsx
className="inline-flex items-center justify-center bg-[var(--accent)]
  text-[var(--text-on-accent)] px-6 py-3 rounded-full text-sm
  font-mono font-bold transition-all duration-300
  hover:shadow-[0_0_20px_var(--accent-glow)] hover:scale-[1.03]"
```

**Nachher:**
```tsx
className="inline-flex items-center justify-center rounded-full
  bg-[var(--accent)] px-6 py-3 text-sm font-mono font-bold tracking-[0.02em]
  text-[var(--text-on-accent)]
  transition-all duration-300 ease-[cubic-bezier(0.16,1,0.3,1)]
  hover:scale-[1.03] hover:shadow-[0_0_20px_var(--accent-glow)]
  active:scale-[0.98]"
```

| Diff | Warum |
|---|---|
| `+ tracking-[0.02em]` | DS §B Typografie: Button = Geist Mono +0.02em letter-spacing |
| `+ ease-[cubic-bezier(0.16,1,0.3,1)]` | DS §E Motion: matches `--ease-out` Token-Wert exakt (Tailwind-default `ease-out` ist cubic-bezier(0,0,0.2,1) — knapp daneben) |
| `+ active:scale-[0.98]` | DS §C Interaction-States: Active = Scale 0.98 (Press-Feedback) |

### Typography

Keine Änderung. H1 hatte bereits `font-mono text-4xl sm:text-5xl lg:text-6xl font-bold leading-[1.05] tracking-tight` — matcht DS §B Skala (Hero H1 = Geist Mono 700, line-height 1.05, tracking -0.02em ≈ `tracking-tight`). Eyebrow + Tagline bereits Geist Mono uppercase tracked-out.
