  <name>Task 1: SignalGrid Canvas Component — Core (Nodes + Idle-Breathing)</name>
  <files>apps/website/components/ui/signal-grid.tsx</files>
  <behavior>
    - Component rendert ein `<div class="relative">` Wrapper mit `<canvas class="absolute inset-0 z-0">` + `<div class="relative z-10">{children}</div>`
    - Canvas fills Parent 100%×100% via `absolute inset-0` + Canvas-Width/Height-Sync an devicePixelRatio (crisp on Retina)
    - Nodes: Grid von ~20×12 auf Desktop, responsive weniger auf Mobile (~12×8). Node = small dot (radius 1.5-2.5px).
    - Idle-Animation: jeder Node hat individuelle Phase-Offset (`Math.random() * Math.PI * 2`) und breathet zwischen opacity 0.15 und 0.35 über ~4-6s pro Zyklus — stagger durch Phase-Offset sorgt für organisches Wirken, nicht im Gleichtakt.
    - Base-Color für Nodes: gelesen aus CSS-Var `--neon-9` (oder passendem DS-Accent-Token) via `getComputedStyle(document.documentElement).getPropertyValue('--neon-9')`.
    - `prefers-reduced-motion`: wenn aktiv, Nodes statisch bei opacity 0.25, keine Breathing-Loop, keine rAF-Animation für Breathing.
  </behavior>
  <action>
    Erstelle `apps/website/components/ui/signal-grid.tsx` als Client-Component (`"use client"`). Strikt TypeScript, kein `any`.

    Props:
    ```ts
    interface SignalGridProps {
      children?: React.ReactNode;
      className?: string;
      nodeCount?: { cols: number; rows: number }; // default: responsive desktop 20×12, mobile 12×8
    }
    ```

    Innere Struktur:
    - useRef<HTMLCanvasElement> + useRef<HTMLDivElement> (container)
    - useState für computed node positions + animation refs
    - useEffect: Resize-Observer → recompute node-grid on container-resize
    - useEffect: rAF-Loop für Breathing-Animation
    - useEffect: prefers-reduced-motion-Media-Query-Listener → pausiert rAF-Loop wenn aktiv
    - CSS-Variables-Reading: beim Mount `getComputedStyle` für `--neon-9` (Node-Color) + `--bg` (für Radial-Fade-Overlay) + `--accent-glow` (für Highlight-Nodes)

    Kein Cursor-Interaktion in Task 1 — kommt in Task 2.

    Wrapper-Struktur:
    ```tsx
    <div ref={containerRef} className={`relative ${className ?? ""}`}>
      <canvas ref={canvasRef} className="absolute inset-0 z-0 pointer-events-none" />

      <div className="relative z-10">{children}</div>
    </div>
    ```

    Canvas-Setup:
    - `canvas.width = rect.width * dpr; canvas.height = rect.height * dpr; ctx.scale(dpr, dpr)` für crisp rendering
    - Node-Data-Struktur: `{ x: number; y: number; phase: number; baseOpacity: number }`

    Performance:
    - IntersectionObserver auf containerRef → wenn `isIntersecting === false`, rAF-Loop pausiert (`cancelAnimationFrame`)
    - rAF-Loop resumed on intersect back

<task type="auto" tdd="false">
  <name>Task 2: Cursor-Propagation + Lines + Radial-Fade-Overlay</name>
  <files>apps/website/components/ui/signal-grid.tsx</files>
  <behavior>
    - Auf `mousemove` (throttled auf rAF-Frame): nearest 5-8 Nodes zu Cursor-Position identifizieren
    - Nodes im Cursor-Radius werden "illuminated": opacity rises to ~0.8, size nudges +20%
    - **Propagation:** Effekt rippelt outward — nearest-layer reagiert sofort, nächste Node-Layer reagiert mit Delay (~50-80ms), Layer 3 mit mehr Delay → 2-3 Layer tief
    - **Linien (Edges):** zwischen illuminated Nodes werden Linien gezeichnet — nearest-neighbor-graph innerhalb eines Radius (z.B. 1.5× grid-spacing), stroke-width ~0.5-1px, color `--neon-9` bei alpha 0.3-0.5
    - **Radial-Fade-Overlay:** CSS-pseudo-Overlay im containerRef (NICHT im Canvas), `radial-gradient(circle at center, transparent 0%, var(--bg) 80%)` oder ähnlich — sorgt dafür dass Text in der Mitte readable bleibt, Grid zum Rand hin faded
    - `prefers-reduced-motion`: kein Mousemove-Listener registriert, keine Propagation, keine Lines gerendert. Nodes bleiben statisch (aus Task 1).
    - `pointer-events-none` auf Canvas bleibt — Cursor wird auf der containerRef-Ebene gesampled (window.mousemove oder container.mousemove)
  </behavior>
  <action>
    Erweitere `signal-grid.tsx`:

    - `useEffect`: mousemove-Listener auf window (throttled auf rAF). Speichert {x, y} in ref.
    - In rAF-Loop (bereits aus Task 1): für jeden Node distanceToMouse berechnen, nearest-N finden, illumination-target setzen. Aktuelle opacity → lerp towards target (smoothing, kein hartes Flacken).
    - **Propagation-Logik:** statt alle illuminated Nodes sofort zu illuminieren, staged:
      - Layer 1 (nearest ~3): activation-delay 0ms
      - Layer 2 (next ~5): activation-delay 60ms
      - Layer 3 (next ~8): activation-delay 120ms
      - Activation-Delay = `performance.now() - cursorMoveTimestamp` compared against delay-threshold.
    - **Lines-Rendering:** nach Node-Rendering, zweiter Pass über illuminated-Pairs: wenn 2 illuminierte Nodes innerhalb `1.5 * gridSpacing` Distanz sind → `ctx.beginPath(); ctx.moveTo(...); ctx.lineTo(...); ctx.stroke()` mit alpha basierend auf der weaker der beiden illumination-values.
    - **Radial-Fade-Overlay:** als zweites absolute-positioned div zwischen Canvas und Children:
      ```tsx
      <div
        className="absolute inset-0 z-[1] pointer-events-none"
        style={{
          background: "radial-gradient(ellipse at center, transparent 0%, transparent 40%, var(--bg) 95%)"
        }}
      />
      ```
      (Ellipse weil Hero tendenziell quer-format ist. Theme-aware weil `var(--bg)` im Light-Mode andere Farbe hat.)
    - `prefers-reduced-motion`: früher Return im mousemove-Handler-useEffect, Overlay bleibt gerendert aber ohne Grid-Animation dahinter — Nodes statisch bei base-opacity.

    **Performance-Discipline:**
    - rAF-Loop eine einzige, kein setInterval
    - Lines-Pass nur wenn mindestens 2 illumination > 0.5 (spart Draw-Calls im Idle)
    - IntersectionObserver-Pause aus Task 1 bleibt
    - Throttle mousemove auf rAF-Tick (nicht jedes Mouse-Event eigener Work-Step)

    **Theme-Awareness-Check:**
    - Re-Read CSS-Vars bei Theme-Change — einfachste Variante: MutationObserver auf `document.documentElement` class-Attribute (dark/light toggle ändert `class` auf `<html>`). Bei Change → re-read `--neon-9` und `--bg`, update cached values.
  </action>
