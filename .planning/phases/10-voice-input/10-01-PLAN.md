Erstelle POST-Endpoint der Deepgram-Token generiert:

1. Pruefe `process.env.DEEPGRAM_API_KEY` existiert (sonst 503)
2. POST an `https://api.deepgram.com/v1/auth/grant` mit Bearer Token
3. Extrahiere `token` aus Response
4. Gib `{ token }` zurueck

Rate-Limiting: Nutze Upstash Redis wie bei anderen API-Routes (Pattern aus `/api/chat` uebernehmen).

Error Handling:
- 503 wenn DEEPGRAM_API_KEY fehlt
- 502 wenn Deepgram antwortet aber Fehler
- 502 bei Network-Error

```typescript
// apps/tools-app/app/api/voice/token/route.ts
import { NextResponse } from 'next/server'
import { rateLimit } from '@/lib/rate-limit' // falls vorhanden

export async function POST(request: Request) {
  // Optional: Rate limiting
  // const ip = request.headers.get('x-forwarded-for') ?? 'unknown'
  // const { success } = await rateLimit(ip, 'voice-token', 10, '1m')
  // if (!success) return NextResponse.json({ error: 'Too many requests' }, { status: 429 })

  if (!process.env.DEEPGRAM_API_KEY) {
    return NextResponse.json({ error: 'Voice service not configured' }, { status: 503 })
  }

  try {
    const res = await fetch('https://api.deepgram.com/v1/auth/grant', {
      method: 'POST',
      headers: {
        Authorization: `Token ${process.env.DEEPGRAM_API_KEY}`,
        'Content-Type': 'application/json',
      },
    })

    if (!res.ok) {
      console.error('[voice/token]', res.status, await res.text())
      return NextResponse.json({ error: 'Token creation failed' }, { status: 502 })
    }

    const data = await res.json()
    return NextResponse.json({ token: data.token })
  } catch (err) {
    console.error('[voice/token] Network error:', err)
    return NextResponse.json({ error: 'Network error' }, { status: 502 })
  }
}
```

WICHTIG: DEEPGRAM_API_KEY muss in `.env.local` und Vercel Env-Vars existieren.

**startRecording():**
1. Pruefe `isSupported` (MediaDevices + MediaRecorder vorhanden)
2. Hole Token via `POST /api/voice/token`
3. Bei Fehler: setError mit deutscher Meldung, return
4. `getUserMedia({ audio: true })`
5. Finde besten mimeType: `audio/webm;codecs=opus` > `audio/webm` > `audio/ogg;codecs=opus` > `audio/mp4`
6. Erstelle WebSocket mit `['token', token]` als Subprotokoll
7. Bei `socket.onopen`: starte MediaRecorder mit 250ms timeslice
8. `mediaRecorder.ondataavailable`: sende Chunks an WebSocket
9. `socket.onmessage`: parse JSON, akkumuliere Transcript

**Transcript-Logik (per D-03):**

socket.onmessage = (event) => {
