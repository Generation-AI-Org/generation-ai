## Approach

Migration in 2 Tasks:
1. **Website kopieren** — Alle relevanten Dateien, package.json anpassen
2. **tools-app kopieren** — Alle relevanten Dateien, package.json anpassen

Danach `pnpm install` im Root und beide Apps testen.

Die Apps werden 1:1 kopiert — keine Code-Änderungen außer package.json name.

## Tasks

### Task 1: Website migrieren

**Goal:** Website-Code ins Monorepo kopieren und lauffähig machen

**Files:**
- `apps/website/` (neu)
- `apps/website/package.json` (anpassen)

**Steps:**

1. Erstelle `apps/website/` Directory-Struktur:
   ```bash
   mkdir -p apps/website
   ```

2. Kopiere diese Ordner von `/Users/lucaschweigmann/projects/GenerationAI/website/`:
   - `app/` (Next.js App Router)
   - `components/`
   - `lib/`
   - `public/`
   - `supabase/` (falls relevant)

3. Kopiere diese Config-Dateien:
   - `next.config.ts`
   - `eslint.config.mjs`
   - `postcss.config.mjs`
   - `components.json`
   - `tsconfig.json`
   - `.prettierrc`

4. Kopiere `package.json` und passe an:
   - `"name": "website"` -> `"name": "@genai/website"`
   - Behalte alle dependencies und scripts
   - Entferne `pnpm-workspace.yaml` falls vorhanden (Root hat das schon)

5. Erstelle `.env.example` basierend auf benötigten Vars:
   ```
   NEXT_PUBLIC_SUPABASE_URL=
   NEXT_PUBLIC_SUPABASE_ANON_KEY=
   ```

6. Kopiere `.env.local` aus Source (für lokale Tests)

**Verify:**
```bash
# Im Root des Monorepos:
ls apps/website/app apps/website/components apps/website/lib
cat apps/website/package.json | grep '"name"'
# Sollte "@genai/website" zeigen
```

**Done:**
- apps/website/ enthält vollständigen App-Code
- package.json hat name "@genai/website"
- Keine node_modules, .next, .git kopiert

### Task 3: Dependencies installieren und testen

**Goal:** Beide Apps lokal starten und verifizieren

**Files:**
- `pnpm-lock.yaml` (wird generiert)
- `apps/website/next.config.ts` (Port 3000)
- `apps/tools-app/next.config.ts` (Port 3001)

**Steps:**

1. Im Root des Monorepos:
   ```bash
   pnpm install
   ```

2. Port-Konfiguration prüfen/anpassen:
   - Website bleibt auf Default (3000)
   - tools-app braucht Port 3001 in package.json dev script:
     ```json
     "dev": "next dev --port 3001"
     ```

3. Website testen:
   ```bash
   pnpm dev:website
   # Browser: http://localhost:3000
   # Sollte Landing Page zeigen
   ```

4. tools-app testen (neues Terminal):
   ```bash
   pnpm dev:tools
   # Browser: http://localhost:3001
   # Sollte Tool-Bibliothek zeigen
   ```

5. Beide gleichzeitig testen:
   ```bash
   pnpm dev
   # Beide Apps sollten parallel starten
   ```

**Verify:**
```bash
# Quick Check ohne Browser:
curl -s http://localhost:3000 | head -20  # HTML Response
curl -s http://localhost:3001 | head -20  # HTML Response
```

**Done:**
- `pnpm install` erfolgreich
- `pnpm dev:website` -> localhost:3000 zeigt Website
- `pnpm dev:tools` -> localhost:3001 zeigt tools-app
- `pnpm dev` -> beide Apps starten parallel
