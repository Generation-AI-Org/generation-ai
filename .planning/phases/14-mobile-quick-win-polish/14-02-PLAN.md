# Plan 14-02: Legal Footer — Hide on Expanded Chat + Darkmode-Farbe (D-03)

## Ziel

Mobile Legal Footer (`AppShell.tsx`, Z. 340–349) soll:
1. Nicht sichtbar sein, wenn Chat mobile full-screen expanded ist.
2. Theme-aware Text-Farbe haben (statt hartem `text-text-muted`).

## Kontext

- Datei: `apps/tools-app/components/AppShell.tsx`
- `isChatExpanded` state existiert bereits (Z. 28), wird an `FloatingChat` via `onExpandChange={setIsChatExpanded}` uebergeben (Z. 336)
- Bestehende Footer-Zeile (Z. 341):
  ```tsx
  <footer className="lg:hidden flex items-center justify-center gap-4 py-2 border-t border-[var(--border)] bg-[var(--bg-header)] text-xs text-text-muted shrink-0">
  ```

## Umsetzung

### Schritt 1: Visibility an `isChatExpanded` koppeln

Footer-`className` um konditionalen `hidden`-Modifier erweitern. Einfachster Weg (kein Re-Mount, kein Layout-Shift-Risiko):

```tsx
<footer
  className={`lg:hidden flex items-center justify-center gap-4 py-2 border-t border-[var(--border)] bg-[var(--bg-header)] text-xs shrink-0 ${
    isChatExpanded ? 'hidden' : ''
  }`}
>
```

**Entscheidung `hidden` vs `opacity-0 pointer-events-none`:** `hidden` gewinnt, weil:
- Footer nimmt keinen Layout-Platz mehr ein (Chat kann voll Hoehe nutzen)
- Keine unsichtbaren Click-Targets
- Trade-off: kein Fade-Transition — fuer Phase 14 (Quick-Win) ok; falls Anim gewuenscht, Phase 16

### Schritt 2: Darkmode-Farbe fixen

`text-text-muted` ersetzen durch das Tailwind-v4 CSS-Variable-Token, das im globalen Theme definiert ist. Token-Wahl haengt vom tatsaechlichen Setup in `app/globals.css` / `@theme` ab — Executor prueft:

- Wenn `--text-muted` CSS-Var definiert und Darkmode sie ueberschreibt: `text-[var(--text-muted)]` → bereits theme-aware
- Wenn nicht: shadcn-aehnliches Token wie `text-muted-foreground` oder Kontrast-Check-Pass

**Executor-Anweisung:** Zuerst `grep -r "text-muted" apps/tools-app/app/globals.css apps/tools-app/tailwind.config.*` um das bestehende Token-System zu verstehen, dann gezielt ersetzen. Der Link-`hover:text-text` (Z. 342, 346) analog pruefen.

### Schritt 3: Visuell verifizieren

- Mobile viewport, Chat expanded → Footer weg
- Mobile viewport, Chat collapsed → Footer sichtbar
- Darkmode toggeln → Footer-Text lesbar (Kontrast WCAG AA)

## Verification (Success Criteria)

1. Mobile, Chat closed: Footer sichtbar, lesbar in Light + Dark.
2. Mobile, Chat expanded: Footer visuell nicht vorhanden, kein Whitespace-Platzhalter.
3. Desktop (lg+): unveraendert (Footer sowieso `lg:hidden`).
4. `pnpm -F tools-app build` + `pnpm lint` gruen.

## Out of Scope

- Anim/Fade-Transition beim Hide/Show (→ Phase 16 Micro-Animations)
- Refactor des globalen Color-Token-Systems
- Footer-Text oder Link-Styling aendern
