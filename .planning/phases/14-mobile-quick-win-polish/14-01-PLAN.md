# Plan 14-01: Auto-Resize fuer Diktat-Input (D-02)

## Ziel

Textarea in `FloatingChat.tsx` soll bei **allen** `message`-Value-Aenderungen (Typing, Diktat, Paste, programmatisch) dynamisch wachsen/schrumpfen — nicht nur bei User-`onChange`.

## Kontext

- Datei: `apps/tools-app/components/chat/FloatingChat.tsx`
- Bestehende Resize-Logik:
  - In `handleInputChange` (ca. Z. 319–326): resized direkt im onChange-Handler
  - Bestehender `useEffect` auf `[message]` (Z. 215–223): versucht es bereits, scheint aber nicht zuverlaessig zu feuern bzw. verwendet inline-Duplikat-Code
- Zwei Textarea-JSX-Pfade (Z. 503 + Z. 878) teilen einen `textareaRef` — nur der aktuell gemountete Pfad haelt das Ref.

## Umsetzung

### Schritt 1: Resize-Logik extrahieren

Die beiden duplizierten Resize-Implementierungen (`handleInputChange` + existierender `useEffect`) durch **eine** wiederverwendbare Funktion ersetzen:

```tsx
const resizeTextarea = useCallback(() => {
  const textarea = textareaRef.current
  if (!textarea) return
  textarea.style.height = 'auto'
  const maxHeight = 120
  textarea.style.height = `${Math.min(textarea.scrollHeight, maxHeight)}px`
}, [])
```

Platzierung: nahe `textareaRef`-Deklaration (ca. Z. 42).

### Schritt 2: useEffect auf [message, isExpanded] umstellen

Bestehenden Effect (Z. 215–223) ersetzen durch:

```tsx
useEffect(() => {
  resizeTextarea()
}, [message, isExpanded, resizeTextarea])
```

Grund fuer `isExpanded` in Deps: beim Wechsel zwischen den beiden Render-Pfaden muss das neu gemountete Textarea seine Hoehe frisch berechnen.

### Schritt 3: handleInputChange vereinfachen

`handleInputChange` (ca. Z. 319+) ruft nur noch `setMessage(value)` auf — der useEffect uebernimmt das Resize. Falls noch anderer Logik drin ist (maxChars etc.), bleibt die.

### Schritt 4: Clear-Pfade pruefen

Stellen verifizieren wo `setMessage('')` gesetzt wird (Z. 259, ggf. nach stopGeneration): Effect feuert automatisch, kein manueller `style.height = 'auto'`-Reset mehr noetig.

## Verification (Success Criteria)

1. **Manuelles Typing:** Textarea waechst beim Tippen wie bisher (max 120px, dann scrollt).
2. **Voice-Input:** Aufnahme starten, sprechen — Textarea waechst live mit dem transkribierten Text.
3. **Clear nach Send:** Nach Nachricht senden ist Textarea wieder auf 44px.
4. **Expanded <-> Collapsed:** Beim Umschalten bleibt Textarea-Hoehe konsistent.
5. **Build/Lint:** `pnpm -F tools-app build` + `pnpm lint` gruen.
