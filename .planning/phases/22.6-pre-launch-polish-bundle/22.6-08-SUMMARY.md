## Decisions Made

- **--fs-h1 token, not --fs-display.** UI-SPEC.md §Typography "tools-app hero specifics" calls for `var(--fs-h1)` (clamp 32→48px) on the tools-app hero — distinct from the website subpage-hero pattern (`var(--fs-display)`, clamp 40→76px). Reason: tools-app hero is INLINE (above the filter+grid stream), not full-viewport. Larger scale would burn vertical real-estate above-the-fold and push tools below the fold on smaller laptops.

- **No Signal-Grid-Echo background in V1.** Decision B-09 says "text-only is the default, optional Signal-Grid-Echo BG only if implementation is clean". Skipped per the principle of "minimal code that solves the problem" (Karpathy/CLAUDE.md rule). Implementation stays at 57 LOC. If Luca wants visual interest later, adding `<SignalGrid nodeCountX={18} nodeCountY={6} aria-hidden className="absolute inset-0 -z-10 opacity-30" />` inside the section is a 1-line append.

- **FilterBar moved INTO the scroll container (not separate sticky wrapper).** Plan's `<action>` block presented two options:
  1. Move FilterBar inside the scroll container (V1: scroll-with-content)
  2. Add `sticky top-0 z-10 bg-bg` to keep it viewport-sticky

  Chose option 1 per CONTEXT.md §B.2 explicit V1 contract ("scroll-with-content is fine — matches website pattern"). Sticky-filterbar can be added later as a 1-line className change if Luca prefers. V1 prioritizes simplicity + matches website pattern of filters scrolling with page content.
