## Decisions Made

- **Client-side auth check (A-07):** Used `createClient()` from `@genai/auth/browser` in a lazy `useEffect` triggered on modal open. Matches the plan's recommendation over a server action — simpler and sufficient for this conversion flow. Auth race (clicking anmelden before getSession resolves) is safe: falls back to logged-out path which sends to /join, where the user gets auto-redirected post-login via Phase 23's redirect_after handling.

- **@base-ui aria-modal behavior:** @base-ui 1.4.0 renders `role="dialog"` on the Popup element rather than setting `aria-modal="true"` as an explicit attribute. The Playwright test was adapted to check `role="dialog"` — semantically equivalent (WAI-ARIA dialog role implies modality). This is a correct and standards-compliant pattern.

- **Modal state at grid level:** All EventCard instances in the grid share one modal instance. Cheaper than per-card state (one portal, one backdrop), and the selectedEvent state retains its value during close-animation allowing smooth fade-out.
