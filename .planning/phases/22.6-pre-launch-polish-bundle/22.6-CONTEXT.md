# Phase 22.6 — Pre-Launch Polish Bundle

> Zwei thematisch verwandte Polish-Tracks zusammengelegt: **Track A** baut die `/events`-Seite (Members-Only-Akquisitionshebel), **Track B** poliert die tools-app (Login-Button, Hero, Nav-Sync). Verschiedene Apps → kein Merge-Konflikt-Risiko, ein gemeinsamer Discuss → Plan → Execute Cycle.

---

## Mission

Vor dem Launch fehlen zwei letzte Conversion-Bausteine:

1. **Events** als zweiter Akquisitionshebel neben `/join` direkt — öffentliche Liste, Anmeldung gated auf Signup
2. **tools-app** muss sich navigatorisch und visuell wie Teil der Hauptwebsite anfühlen — aktuell Brüche (alter Login-Button, fehlender Hero, Nav-Drift)

Beide sind Polish, nicht Architektur-Phasen — keine grundsätzlichen Trade-offs zu klären, sondern Wording, Layout-Details, Schema-Felder.

---

## Track A — `/events` Seite (ehemals Phase 22.5)

### Strategische Klärung

Simons Konzept §5.6: **„Unsere Events sind exklusiv für Mitglieder."** Events sind Akquisitionshebel für Signup, nicht Public-Discovery-Ziel. Architektur radikal vereinfacht:

- Keine Circle-Public-Events-Experimente
- Keine Circle-API-Integration
- Kein Luma-Circle-Hybrid
- Event-Hosting bleibt flexibel pro Event (Circle-internal, Luma, …)
- Unsere `/events`-Seite ist nur **Discovery- und Conversion-Kanal** auf der Hauptdomain

### Mission Track A

Besucher:innen sehen auf öffentlicher `/events`-Seite welche Events anstehen. Klick auf „Anmelden" im Modal:
- Logged in → Redirect zur `ctaUrl` des Events (Circle/Luma/etc.)
- Not logged in → Redirect zu `/join?redirect_after=/events/[slug]` (Signup-Gate)

Archiv = minimaler Social Proof, keine Modals.

### Scope Track A

**In-scope:**
- Hero-Sektion (Konzept §5.3): Section-Header „Generation AI · Events" + H1 „Events, die dich weiterbringen." + Subhead
- **Kommende Events** als 3-Kachel-Grid
  - Metadaten: Titel, Datum+Uhrzeit, Format-Tag (Workshop/Speaker/Masterclass), Level-Tag (Einsteiger/Fortgeschritten/Expert), Ort (Online/Stadt/hybrid), Partner (nur bei Masterclasses), Anmelde-Button
  - Bei >3 Events: „Mehr anzeigen ↓" Button (Client-State, keine separate Page)
- **Event-Modal** bei Kachel-Klick: Beschreibung, Agenda, Speaker-Info, Anmelde-Button. Optional ICS-Download.
- **Anmelde-Flow:** Auth-Check → Redirect zur `ctaUrl` (Member) bzw. zu `/join?redirect_after=/events/[slug]` (Non-Member)
- **Formate-Sektion** §5.5: 3 Kacheln (Workshops / Speaker Sessions / Masterclasses)
- **Members-Only-Hinweis** §5.6: erklärt Signup-Gate („Kostenlos, 2 Minuten")
- **Archiv-Sektion** §5.7: minimale Kacheln (Foto + Titel + Datum + 2-Satz-Recap), keine Modals
- **Abschluss-CTA** §5.8: „Keine Events verpassen. [Jetzt beitreten]"
- **MDX-Pipeline** in `apps/website/content/events/`:
  ```
  apps/website/content/events/
  ├── 2026-05-15-prompting-masterclass.mdx
  ├── 2026-05-28-agents-workshop.mdx
  └── past/
      └── 2026-03-10-launch-event.mdx
  ```
- Frontmatter:
  ```yaml
  title: string
  date: ISO-DateTime
  format: "Workshop" | "Speaker Session" | "Masterclass"
  level: "Einsteiger" | "Fortgeschritten" | "Expert"
  location: "Online" | string | "hybrid"
  partner: string | null
  ctaUrl: string
  slug: string
  speakers: Array<{name, role, avatar?}>
  ```
- Build-Logic: `date >= now` → Kommende, `date < now` → Archiv
- Event-Detail-URL `/events/[slug]` für SEO + Share-Links (Modal-Content als Standalone-Page)

**Out-of-scope:**
- Circle-API-Integration, Luma-Embed, ICS-Download (TBD pro Plan), Filter, Search, Email-Benachrichtigungen, Analytics, finales Wording (Phase 27)

### Decisions Track A

- **A-01** — Events sind members-only per Strategie (Simon §5.6). Seite public, Anmeldung gated.
- **A-02** — MDX-Files als Content-Source (Pattern wie Phase 26 `/community`-Artikel reusen — `lib/mdx/reader.ts` + `getAllArticles()`-Pattern auf Events portieren)
- **A-03** — Event-Hosting flexibel via `ctaUrl` Frontmatter. Kein Lock-in.
- **A-04** — Modal zeigt Event-Details öffentlich. Nur Anmeldung gegated. Unentschlossene brauchen Detail für Signup-Motivation.
- **A-05** — Event-Detail als eigene URL `/events/[slug]` zusätzlich zum Modal (SEO + Permalinks). Modal = UX-Preview.
- **A-06** — Archiv ohne Modal. 2-Satz-Recap + Foto reicht.
- **A-07** — Auth-Check Server-side bevorzugt (Route-Handler bei Anmelde-Klick), Client-side mit Supabase-Session als Alternative — TBD im Plan.
- **A-08** — Signup-Gate via Query-Param `?redirect_after=/events/[slug]`, integriert mit Phase 23 `/join`.
- **A-09** — „Mehr anzeigen" als Client-State, keine separate Unterseite.

### Success Criteria Track A

- [ ] `/events` Route in `apps/website/app/events/page.tsx`
- [ ] MDX-Pipeline liest aus `apps/website/content/events/`
- [ ] Kommende Events als 3-Kachel-Grid, chronologisch aufsteigend
- [ ] „Mehr anzeigen" funktioniert bei >3 Events
- [ ] Modal mit Focus-Trap, Escape, aria-modal
- [ ] Anmelde-Flow: Non-Member → `/join?redirect_after=...`, Member → `ctaUrl`
- [ ] `/events/[slug]` rendert als Standalone-Page
- [ ] Archiv minimal (Foto + Titel + Datum + Recap)
- [ ] Members-Only-Hinweis + Abschluss-CTA → `/join`
- [ ] Lighthouse `/events` ≥ 90 in allen 4 Kategorien
- [ ] Mobile: Kacheln stacken, Modal als Full-Screen-Sheet
- [ ] Placeholder-MDX-Files mit „Beispiel"-Badge bis echte Events vorliegen

### Offene Fragen Track A (für Discuss-Phase)

1. **Placeholder-Event-Count:** 3 kommende + 2 Archiv ausreichend?
2. **ICS-Download V1:** ja/nein? (Empfehlung: nein, einfacher Scope)
3. **Event-Fotos:** AI-generierte Header bis echte Fotos da sind? Oder generischer Stock?
4. **Redirect-Param:** `?redirect_after=` (konsistent mit Phase 23) oder `?next=` (Supabase-Konvention)?
5. **MDX-Stack:** Reuse `lib/mdx/reader.ts` aus Phase 26, oder eigene events-Pipeline? (Empfehlung: reuse mit kleinem Schema-Adapter)

## Track B — Tools-Subdomain Polish (ehemals Phase 22.7)

### Mission Track B

tools-app ist funktional solide (Filter + Kacheln + Agent-Chat + Lite/Pro). Aber laut Simon §6.1 muss sie sich **navigatorisch und visuell wie Teil der Hauptwebsite anfühlen** — drei konkrete Brüche:

1. ~~Logo-Link in Nav zeigt auf Community-Subdomain~~ ✅ **Bereits gefixt** in Phase 26 Drive-by Commit (2026-04-25)
2. **Single „Anmelden"-Button** macht Pro-Features unsichtbar und verschleudert Signup-Conversion
3. **Tools-Hero** fehlt → Einsteiger verstehen nicht sofort was sie hier haben

### Scope Track B

**In-scope:**

#### B.1 Login-Button-Umbau (§6.4)
- Aktuell: einzelner „Anmelden"-Button oben rechts
- Neu (logged-out State):
  - **Primary:** „Kostenlos registrieren" → `https://generation-ai.org/join`
  - **Secondary:** „Bereits Mitglied? → Einloggen" → bestehende tools-app Login-Route
- Eingeloggt: User-Menu wie bisher (kein Regression)
- Mobile: Primary in Burger-Menü, Secondary als Text-Link darunter

#### B.2 Tools-Hero (§6.3)
Neue Hero-Sektion über den Filter-Tabs:
```
// deine ki-tool-bibliothek

# KI-Tools, kuratiert für dich.

Über 100 Tools, sortiert nach Anwendungsfall.
Brauchst du Hilfe? Frag unseren Agenten.
```
- Kein langer Marketing-Block, nur diese 3 Zeilen
- DS-Typography (Geist Mono für H1, Sans für Body)
- Zwischen Header-Nav und Filter-Tabs
- Mobile: Padding reduziert, Text bleibt lesbar

#### B.3 Sticky-Nav-Sync-Check
- Hauptdomain hat Sticky-Nav (Phase 20)
- tools-app Nav: verifizieren dass identisch sticky, gleiche Background-Transition, gleiche Höhe
- Falls Drift: Component-Harmonisierung (`packages/ui` oder Code-Sync)

#### B.4 Nav-Struktur-Alignment mit Hauptdomain (§6.1)
- Gleiche Nav-Items: `Events · Tools · Community · Für Partner · Über uns | Jetzt beitreten`
- Tools aktiv markiert solange auf Tools-Subdomain
- Andere Items → Hauptdomain-Routen (`generation-ai.org/events` etc.), gleicher Tab
- „Jetzt beitreten" → `generation-ai.org/join`

**Out-of-scope:**
- Redesign Tool-Kacheln, Filter-Tabs, Agent-Chat, Lite/Pro-Logik (alles funktional ok)
- Tool-Detailseiten Redesign (deferred)
- Suchfeld, erweiterte Pro-Filter (Roadmap)
- Kategorien-Konsistenz mit Hauptdomain-Labels (deferred)

### Decisions Track B

- **B-01** — Hard Domain-Trennung. tools-app bleibt eigene App, eigene Deploys, aber visuell+navigatorisch nahtlos.
- **B-02** — Shared Nav-Component: TBD im Plan (Optionen: `packages/ui`-Component vs. parallele Komponenten in beiden Apps). Trade-off: langfristige Wartbarkeit vs. Scope-Größe.
- **B-03** — „Kostenlos registrieren" als Primary-CTA bestätigt von Luca (2026-04-23).
- **B-04** — ~~Logo-Link-Fix als atomic single-line commit~~ ✅ erledigt (Phase 26 merge).
- **B-05** — Cross-Domain-Nav: Klicks verlassen Subdomain → voller Page-Load. Kein SPA-Routing. Akzeptabel.
- **B-06** — „Jetzt beitreten" cross-domain ohne State-Übertragung (logged-out User haben keinen State).
- **B-07** — Hero-Copy folgt Konzept exakt, finale Wording-Iteration in Phase 27.

### Success Criteria Track B

- [x] ~~Logo-Link führt auf `generation-ai.org`~~ (bereits done)

- [ ] „Kostenlos registrieren" → `https://generation-ai.org/join`
