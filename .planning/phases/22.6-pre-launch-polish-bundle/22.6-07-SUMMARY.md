## Decisions Made

- **Branching pattern: existing `mode === 'public'` prop, not new useAuth() import.** The plan's frontmatter suggested wiring `useAuth()` for logged-in vs logged-out branching. The current GlobalLayout already branches via `mode: ChatMode` prop (public = no auth required = effectively logged-out). The `mode` value is set by parent route based on auth state. Adding `useAuth()` would introduce a second source-of-truth and risk hydration mismatches if the two diverge. Kept the established pattern — semantically equivalent, zero refactor risk. (Rule 3-style guard: blocking change to plan recommendation prevented; no architectural impact; Rule 4 not triggered as this is a low-level pattern choice not a structural one.)

- **Secondary link visibility on narrow screens.** Used `hidden sm:inline-block` on the Secondary link so it shows on default Playwright viewport (1280px) and standard phones (≥640px), but hides on very narrow viewports to prevent the Primary CTA + Secondary text from wrapping awkwardly with the Theme Toggle. Plan 09 will add the proper mobile burger sheet with full-width Primary + below-Primary Secondary stacked, replacing the V1 hide-on-narrow approach.

- **Kept the entire `mode === 'public'` branch as the wrapper for the new CTAs** rather than restructuring the conditional. This minimizes the diff (one block replaced, conditional structure preserved) and keeps the file's existing rhythm intact.
