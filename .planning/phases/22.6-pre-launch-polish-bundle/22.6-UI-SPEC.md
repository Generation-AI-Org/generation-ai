## Spacing Scale

All spacing is driven by `--space-*` tokens from `packages/config/tailwind/base.css`. Use Tailwind's `p-*`/`gap-*`/`m-*` utilities which map to these.

| Token | Value | Usage |
|-------|-------|-------|
| --space-1 | 4px | Icon gaps, dot separators, micro-padding |
| --space-2 | 8px | Compact internal padding (tag inner, badge inner) |
| --space-3 | 12px | Dense list gaps, secondary button padding |
| --space-4 | 16px | Default element spacing, card inner padding (mobile) |
| --space-6 | 24px | Section inner padding, card inner padding (desktop) |
| --space-8 | 32px | Gap between cards in grid |
| --space-12 | 48px | Major section top/bottom padding |
| --space-16 | 64px | Hero vertical padding (`py-20` = 80px, acceptable exception per established subpage pattern) |
| --space-24 | 96px | Section-to-section rhythm on large screens |

Exceptions:
- Hero `py-20` (80px) is the established subpage pattern from `/community` — kept for consistency, not changed.
- `12px (--space-3)` — Tailwind p-3 default used for secondary-button inner-padding + dense list gaps. 8px reads cramped on tags, 16px reads too loose. Established Tailwind/DS convention reused across all subpages.
- `96px (--space-24)` — large-viewport (1440px+) section-to-section rhythm. 64px feels cramped at desktop widths on hero+sections layout. Used in Phase 21 /about + Phase 26 /community.
- Modal `p-4` on mobile, `p-6` on desktop — matches card inner rhythm.
- Touch targets: minimum 44px height for all interactive elements (A11y WCAG 2.1 AA).

## Typography

All sizes reference DS tokens. No inline `clamp()` in component code — use tokens.

### Primary Scale (4 sizes, 2 weights)

This phase reuses the established Generation AI Design System (live in production since Phase 16). The primary scale covers the four visible hierarchy roles:

| Role | Size | Weight | Line Height | Font | Letter Spacing |
|------|------|--------|-------------|------|----------------|
| Hero H1 (events, tools hero) | `var(--fs-display)` = clamp(40px, 6.5vw, 76px) | 700 | 1.02 | Geist Mono | -0.03em |
| Section H2 | `var(--fs-h2)` = clamp(24px, 3vw, 32px) | 700 | 1.2 | Geist Sans | -0.015em |
| Body / Card body / Modal description | 16px | 400 | 1.65 | Geist Sans | 0 |
| Eyebrow / Tags / Captions (all micro-text) | 11px | 700 | 1.3 | Geist Mono | +0.2em eyebrow / +0.08em tags |

**Primary weights:** 400 (Geist Sans body) and 700 (Geist Mono — inherently bold via font choice, AND emphasized Sans elements).

### Declared Exceptions (existing DS patterns — no new tokens)

The following sizes appear in this phase because they are established DS tokens already shipped in production subpages (/community, /about, /partner, /events). Removing them would break parity with Phase 26 patterns this phase mirrors.

| Size / Weight | Justification |
|---------------|---------------|
| `20px (var(--fs-h3))` | Established DS token used for card titles in /community, /partner, /events. Reuse keeps subpage parity with Phase 26 card grid pattern. |
| `18px lede` | Established DS subhead pattern from Phase 21 /about + Phase 26 /community heroes. Used as `text-lg sm:text-xl` for visual rhythm below H1. |
| `14px button / body-small` | Geist Mono button-label scale per brand/DESIGN.md DS-Spec. Also used for archive card recap text — 16px reads too heavy for secondary content. |
| `13px caption/meta` | Date and location meta rows in event cards. Lighter than body-small to visually recede; established pattern in DS card components. |
| Weight 600 | Intermediate Geist Sans weight for card-title emphasis (20px h3 role). DS-defined intermediate. Removal would force flat hierarchy on Phase-26 patterns this phase mirrors. |
| Weight 500 | Intermediate Geist Sans weight for lede subheads (18px). DS-defined intermediate. Removal would flatten the hero/subhead hierarchy established in Phases 21 + 26. |

**Tools-app hero specifics (B.2):**
- Label `// deine ki-tool-bibliothek`: Geist Mono, 11px, weight 700, uppercase, tracking-[0.2em], color text-text-muted
- H1 `KI-Tools, kuratiert für dich.`: `var(--fs-display)` or `var(--fs-h1)` — use `--fs-h1` (clamp 32px→48px) for the tools hero since it sits inline (not full-viewport), `font-mono font-bold leading-[1.02] tracking-[-0.03em]`
- Body: Geist Sans 16px/1.65, color text-text-secondary

## Color

Dual-theme system. All values are theme-aware CSS variables — do not hardcode hex in components.

| Role | Dark Token / Value | Light Token / Value | Usage |
|------|--------------------|---------------------|-------|
| Dominant surface (60%) | `--bg: #141414` | `--bg: #F6F6F6` | Page background, modal backdrop base |
| Secondary surface (30%) | `--bg-elevated` (slate-2 dark) | `--bg-elevated` (slate-2 light) | Cards, event grid cards, format cards, archive cards, tools-app header bg |
| Accent (10%) | `--accent: #CEFF32` (neon) | `--accent: #F5133B` (red) | Eyebrow dot, active node glow, primary CTA background, "Zum Event anmelden" button, format tag border-on-hover |
| Border | `--border: rgba(255,255,255,0.08)` | `--border: rgba(0,0,0,0.1)` | Card borders, modal border, input border |
| Header band | `--bg-header` (blue #3A3AFF dark / pink #FC78FE light) | same | Sticky nav background (both apps) |
| Destructive | `#DC2626` (semantic, not brand) | same | Non-Member "Anmelden" redirect warning if needed |

**Accent reserved for:**
- Primary CTA button fill (`bg-[var(--accent)]`)
- Eyebrow accent dot in hero sections
- Format-Tag / Level-Tag accent border on hover
- `BeispielBadge` fill (theme-aware: brand-neon dark / brand-red light — already implemented)
- Members-Only-Hinweis left-border stripe
- SignalGrid activated node color (canvas only, no Tailwind class)
- "Kostenlos registrieren" primary button fill in tools-app nav

**Accent NOT used for:** body text, card backgrounds, section backgrounds, borders (except accent-border variant), archive card imagery frames.

**Text tokens:**
- `text-text` — primary text (headings, body)
- `text-text-muted` — secondary/meta text (date, location, eyebrow label)
- `text-text-secondary` — midtone body text (tools hero body, subhead below H1)
- `text-on-accent` — text on accent-filled buttons (`#141414` dark / `#FFFFFF` light)

**Source:** brand/DESIGN.md §C, base.css lines 99-257, signal-grid.tsx color system.

## Animation Contract

All animations require `useReducedMotion` guard. If `prefersReducedMotion === true`: skip all `initial`/`animate` props entirely (pass `false` to `initial`).

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Hero H1 + content | `opacity: 0→1, y: 16→0` | 0.7s | `easeOut` (matches community-hero) |
| Event cards (on "Mehr anzeigen" expand) | `opacity: 0→1, y: 8→0` | 0.3s | `var(--ease-out)` |
| Modal backdrop | `opacity: 0→1` | `var(--dur-normal)` = 300ms | linear |
| Modal popup | `opacity: 0→1, y: 16→0` | `var(--dur-normal)` = 300ms | `var(--ease-out)` |
| Tools-app hero | `opacity: 0→1, y: 12→0` | 0.5s | `easeOut` |
| Nav hover transitions | `color`, `border-color` | `var(--dur-fast)` = 150ms | linear |
| Button hover (`scale`, `shadow`) | transform + shadow | `var(--dur-fast)` = 150ms | linear |
| SignalGrid background | Canvas RAF loop — reduced-motion: frozen static nodes | — | — |

CSS `@media (prefers-reduced-motion: reduce)` fallback must be applied to any CSS `animation` or `transition` outside of React components (e.g., button hover via Tailwind class).

## Accessibility Contract

WCAG 2.1 AA mandatory throughout.

| Requirement | Implementation |
|-------------|----------------|
| Skip link | Existing site-wide skip link to `#main-content` (already implemented) |
| Focus-visible | All interactive elements: `outline-2 outline-offset-2 outline-[var(--accent)]` |
| Event card keyboard | `role="button"` + `tabIndex={0}` + `onKeyDown` Enter/Space → opens modal |
| Modal aria | `Dialog.Popup` has `aria-modal="true"` (native @base-ui), `Dialog.Title` required |
| Modal focus | Focus moves to modal on open; returns to trigger on close (native @base-ui) |
| Modal ESC | Closes modal (native @base-ui) |
| Modal close button | Explicit `aria-label="Event schließen"` on the icon-only `Dialog.Close` trigger |
| Archive cards | `role="article"` or `<article>` semantic element; no interactive elements = no keyboard trap concern |
| Images | Event header images: `alt="[Event title]"` — descriptive. AI-generated images: `alt="[Event title] — Vorschaubild"`. Brand-pattern fallback: `aria-hidden="true"` (decorative) |
| SignalGrid canvas | `aria-hidden="true"` (decorative canvas, not conveying information) |
| Color contrast | Accent on dark bg `#CEFF32` on `#141414`: 12.6:1 (AAA). Accent on light bg `#F5133B` on `#F6F6F6`: 4.7:1 (AA). `text-text-muted` must maintain ≥ 4.5:1 against bg |

## Platform Constraints (LEARNINGS.md)

These are hard rules derived from production incidents.

| Constraint | Rule |
|------------|------|
| CSP Nonce | `app/events/page.tsx` MUST call `await headers()` to get the nonce — even if not using it directly, this makes the route dynamic |
| Force-dynamic check | After `pnpm build`, `/events` and `/events/[slug]` must show `ƒ` (dynamic), NOT `○` (static). Static = no nonce = CSP blocks all scripts = black page |
| Nonce on request headers | `proxy.ts` sets nonce on request headers, not response — inherited by Root Layout. No changes to proxy.ts needed in this phase |
| @base-ui Dialog | Use `Dialog.Portal` + `Dialog.Backdrop` + `Dialog.Popup` — NOT native `<dialog>` HTML element. Pattern verified in `sheet.tsx` |
| tools-app h-screen layout | Hero-Sektion must be inside the `overflow-y-auto` scroll container in HomeLayout.tsx, not outside it — otherwise content is clipped |
| Cross-domain links | All links from tools-app to generation-ai.org must be `<a href="...">` NOT `<Link>` — full page load, same tab, no `target="_blank"` |
| Reduced-motion | `useReducedMotion()` guard on ALL motion/react animations. CSS-only animations need `@media (prefers-reduced-motion: reduce)` fallback |
| Umlaute | All user-facing copy: ö/ä/ü/ß (never oe/ae/ue/ss). MDX files, component strings, copy constants — all must use correct characters |
