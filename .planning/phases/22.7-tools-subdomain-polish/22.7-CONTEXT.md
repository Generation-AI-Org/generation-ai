# Phase 22.7 — Tools-Subdomain Polish & Nav-Integration

> Reaktive Fixes auf der bereits live laufenden `tools.generation-ai.org` — Login-Button differenzieren, Logo-Link-Bug fixen, Hero-Text ergänzen, Sticky-Nav-Sync mit Hauptdomain sicherstellen. Umsetzung nach Simons Konzept §6.

---

## Mission

Die tools-app ist funktional bereits solide (Filter + Kacheln + Agent-Chat + Lite/Pro-Logik laufen). Aber laut Simon §6.1 muss sie sich **navigatorisch und visuell wie Teil der Hauptwebsite anfühlen** — aktuell sind drei konkrete Brüche da, die den Eindruck zerstören:

1. **Logo-Link in Nav zeigt auf Community-Subdomain** statt Hauptdomain → User landet in falscher Welt
2. **Single „Anmelden"-Button** macht Pro-Features unsichtbar und verschleudert Signup-Conversion
3. **Tools-Hero** fehlt → Einsteiger verstehen nicht sofort was sie hier haben

Diese Phase fixt genau das, ohne die funktionierende Kacheln/Filter/Chat-Logik anzutasten.

---

## Scope

**In-scope:**

### 1. Logo-Link-Fix (CRIT)
- `<Logo />` im Header der tools-app: Link geht aktuell auf `community.generation-ai.org` (vermutlich Copy-Paste-Error oder Altlast)
- Ziel: Link auf `generation-ai.org` (Hauptdomain)
- Verifikation: auf Preview klicken, landet auf Hauptdomain

### 2. Login-Button-Umbau in Nav-Header (§6.4)
- Aktueller Zustand: einzelner „Anmelden"-Button oben rechts
- Neuer Zustand:
  - **Primary Button:** `Kostenlos registrieren` → `https://generation-ai.org/join`
  - **Secondary Link** (daneben oder darunter): `Bereits Mitglied? → Einloggen` → bestehende Login-Route der tools-app
- Für eingeloggte User: beide Varianten weg, stattdessen User-Menu (wie bisher)
- Mobile: Primary-Button in Burger-Menü, Secondary als Text-Link darunter

### 3. Tools-Hero (§6.3)
- Neue Hero-Sektion über den Filter-Tabs:
  ```
  // deine ki-tool-bibliothek

  # KI-Tools, kuratiert für dich.

  Über 100 Tools, sortiert nach Anwendungsfall.
  Brauchst du Hilfe? Frag unseren Agenten.
  ```
- Kein langer Marketing-Block, nur diese 3 Zeilen
- DS-Typography (Geist Mono für H1, Sans für Body)
- Zwischen Header-Nav und Filter-Tabs
- Mobile: Padding reduziert, Text bleibt lesbar

### 4. Sticky-Nav-Sync-Check
- Hauptdomain hat Sticky-Nav (ab Phase 20 Skeleton)
- tools-app Nav: verifizieren dass sie identisch sticky ist
- Visuelles Scroll-Verhalten identisch (gleiche Background-Transition, gleiche Höhe)
- Falls Drift: Component-Harmonisierung mit Hauptdomain (shared in `packages/ui` oder duplizierter Code synchronisieren)

### 5. Nav-Struktur-Alignment mit Hauptdomain
- Laut §6.1: gleiche Nav-Items wie Hauptdomain (Events · Tools · Community · Für Partner · Über uns | Jetzt beitreten)
- Tools ist **aktiv markiert** solange man auf Tools-Subdomain ist
- Andere Items führen auf entsprechende Hauptdomain-Routen (`generation-ai.org/events` etc.)
- Klicks öffnen im **gleichen Tab**, nicht neue Tabs
- „Jetzt beitreten"-CTA in tools-app-Nav führt auf `generation-ai.org/join`

**Out-of-scope:**
- Redesign der Tool-Kacheln oder Filter-Tabs (funktional ok)
- Agent-Chat-Änderungen (bleibt wie live)
- Lite/Pro-Logik-Änderungen (bereits funktional)
- Tool-Detailseiten Redesign (deferred, eigenes Projekt)
- Suchfeld, erweiterte Filter für Pro (Roadmap)
- Kategorien-Konsistenz mit Hauptdomain-Labels (z.B. „Bildgenerierung" vs „Bilder & Design") — deferred, separater Task

## Decisions

- **D-01** — Hard Domain-Trennung: tools-app bleibt technisch auf tools.generation-ai.org (eigene App, eigene Deploys), aber visuell und navigatorisch nahtlos zur Hauptdomain.
- **D-02** — Shared Nav-Component? Zwei Optionen:
  - (a) Component in `packages/ui` oder neuem `packages/nav` → beide Apps importieren
  - (b) Parallele Komponenten in beiden Apps, manuell synchron halten
  - Empfehlung: (a) für langfristige Wartbarkeit, aber (b) akzeptabel wenn Scope klein bleibt. TBD im Plan.
- **D-03** — „Kostenlos registrieren" als Primary-CTA laut Luca OK (bestätigt 2026-04-23).
- **D-04** — Logo-Link-Fix ist Single-Line-Change, aber kritisch → eigener Commit, atomic.
- **D-05** — Cross-Domain-Navigation: Klicks auf „Events", „Community", „Für Partner", „Über uns" verlassen die Subdomain → voller Page-Load auf Hauptdomain. Kein SPA-Routing. Das ist akzeptabel.
- **D-06** — „Jetzt beitreten" muss cross-domain zu `/join` führen, State-Übertragung? Nein — bei nicht-eingeloggten Usern gibt's keinen State zu übertragen. Redirect reicht.
- **D-07** — Hero-Copy folgt Simons Konzept exakt, finale Wording-Iteration in Phase 27 Copy-Pass.
