### Task 1 — PlaceholderAvatar Component

**Read first:**

**Action:**
Erstelle `apps/website/components/about/placeholder-avatar.tsx` mit:

```tsx
type PlaceholderAvatarProps = {
  name: string          // Full name, used to derive initials
  size?: "sm" | "md" | "lg"  // sm=40px, md=80px, lg=120px
  className?: string
}

export function PlaceholderAvatar({ name, size = "md", className }: PlaceholderAvatarProps) {
  // Derive initials: "Janna Meister" → "JM" (first letter of first two words, fallback single letter)
  // Size map: sm → "h-10 w-10 text-sm", md → "h-20 w-20 text-xl", lg → "h-28 w-28 text-3xl" (tailwind spacing scale, 4px base — 10=40px, 20=80px, 28=112px close to 120)
  // Classes: aspect-square rounded-full bg-bg-elevated border border-border flex items-center justify-center font-mono font-bold text-text tabular-nums
  // aria-hidden NICHT setzen — Consumer setzt aria-label auf Wrapper ("Platzhalter-Porträt von {name}")
}
```

- Server Component (kein `'use client'` nötig — pure render, keine Hooks)
- Initialen-Algorithmus: Split name by whitespace, take first letter of first two tokens, `.toUpperCase()`. Fallback: Erste 1-2 Chars von name wenn kein Leerzeichen.
- Size-Map exakt: `sm`=40px (members bei Mobile), `md`=80px (members Desktop), `lg`=112px (founders — close to SPEC-Empfehlung 120px, aber Tailwind-Scale-konform — alternativ `size-[120px]` inline)
  - **Entscheidung:** `lg`=`h-[120px] w-[120px] text-[28px]` (exakt SPEC), `md`=`h-20 w-20 text-xl` (80px), `sm`=`h-10 w-10 text-sm` (40px)
- Text-Color: `text-text` (primary), KEIN accent
- Background: `bg-bg-elevated`
- Border: `border border-border`

### Task 2 — TeamMemberCard Component

**Read first:**
- `apps/website/components/about/placeholder-avatar.tsx` (eben erstellt in Task 1)
- `apps/website/components/sections/kurz-faq-section.tsx` (für Hover-Transition-Pattern: `transition-colors duration-[var(--dur-normal)] ease-[var(--ease-out)] hover:border-[var(--border-accent)]`)

**Action:**
Erstelle `apps/website/components/about/team-member-card.tsx` mit:

```tsx
import { PlaceholderAvatar } from "./placeholder-avatar"

type TeamMemberCardProps = {
  name: string
}

export function TeamMemberCard({ name }: TeamMemberCardProps) {
  // Layout: vertical flex (avatar centered, name below), text-center
  // Container: rounded-2xl bg-bg-card border border-border px-4 py-6 flex flex-col items-center gap-3
  // Hover: transition-all duration-[var(--dur-normal)] ease-[var(--ease-out)] hover:border-[var(--border-accent)] hover:-translate-y-0.5 (dezentes Lift)
  // Reduced-motion: useReducedMotion() in einem Client-Wrapper — ABER: Card selbst ist präferabel Server-Component (kein State). Hover via CSS-only (transform + border). CSS `prefers-reduced-motion` in globals.css greift auf `transition-all`.
  //   → Entscheidung: Server Component, Hover-State rein via CSS classes. Kein JS-Hook nötig.
  // Avatar: <PlaceholderAvatar name={name} size="md" />
  // Name-Label: <span className="font-sans text-[13px] font-normal text-text-secondary text-pretty">{name}</span>
  // aria: Wrapper hat kein role, PlaceholderAvatar wird in einem Fragment gerendert — name ist screenreader-visible via <span>
}
```

- Server Component (kein `'use client'`)
- Hover-Animation rein via Tailwind `transition-all` + `hover:-translate-y-0.5 hover:border-[var(--border-accent)]` (motion-safe handled by global CSS `prefers-reduced-motion: reduce` → disables transition)
- aria-label auf Avatar-Wrapper: `aria-label={"Platzhalter-Porträt von " + name}` (wenn Foto-Swap Phase 27 echtes Foto rendert, wird das alt-text)
- Name: `<span className="font-sans text-[13px] font-normal text-text-secondary text-pretty">`

| Threat | Surface | Mitigation |
