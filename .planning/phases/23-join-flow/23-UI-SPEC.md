### Font Assignment by Element

| Element | Font | Rationale |
|---------|------|-----------|
| Hero Eyebrow (`// jetzt beitreten`) | Geist Mono | Brand signature, uppercase micro treatment |
| Hero H1 | Geist Mono | DS §B: H1 = Geist Mono 700, global `h1` rule in base.css |
| Benefit-Icons pattern row | Geist Mono | Micro/tag treatment, same as eyebrow |
| Form field labels | Geist Mono | Consistent with partner-contact-form.tsx established pattern |

| Checkbox labels | Geist Sans | Body-adjacent, readable |
| Submit button | Geist Mono | DS §B: buttons = Geist Mono 700 letter-spacing 0.02em |
| Success headline | Geist Sans | H2 role: section headings = Geist Sans per DS §B |
| Success body copy | Geist Sans | Body role |
| Lede / intro paragraph | Geist Sans | Body role at `--fs-body`, `leading-[1.5]`, `text-pretty` |

### Type Scale

Exactly 4 canonical sizes. Error messages are a sub-role annotation within role #3, not a 5th size.

| # | Role | Token / Value | Weight | Line-Height | Font | Letter-Spacing | Elements |
|---|------|--------------|--------|-------------|------|----------------|---------|
| 1 | Hero Display | `var(--fs-display)` = `clamp(40px, 6.5vw, 76px)` | 700 | `leading-[1.02]` | Geist Mono | `tracking-[-0.03em]` | `<h1>` in hero |
| 2 | Section Head | `var(--fs-h2)` = `clamp(24px, 3vw, 32px)` | 700 | `var(--lh-headline)` = 1.2 | Geist Sans | `-0.015em` | Success card `<h2>` |
| 3 | Body | `var(--fs-body)` = 16px | 400 | `leading-[1.65]` | Geist Sans | default | Form inputs, Intro-Lede (`leading-[1.5]` + `text-pretty`), checkbox labels, success body, secondary link ¹ |
| 4 | Micro | `var(--fs-micro)` = 11px | 700 | 1.4 | Geist Mono | `tracking-[0.2em]` + uppercase | Eyebrow pill, form labels, benefit row items, submit button label, success CTA label, OPTIONAL badge (at `opacity-60` to visually recede) |

**¹ Body sub-role — Error messages:** Inline field errors render at `text-sm` = 14px (Geist Sans, weight 400, lh 1.55). This is a body-error subvariant within role #3: slightly smaller than body for visual subordination while remaining in the body semantic role. Not a 5th canonical size.

**text-wrap:** `text-balance` on H1 and success headline. `text-pretty` on lede and body paragraphs. This matches base.css global rules.

## Layout Contract

### Page Structure

```
<JoinClient nonce={nonce}>           ← 'use client', MotionConfig
  <Header />
  <main id="main-content" className="min-h-screen pt-20">
    <JoinHeroSection />              ← min-h-[60vh], no SectionTransition before
    <JoinFormSection />              ← Form Card + Submit + Success-Swap
    <Footer />                       ← included here, no SectionTransition after form
  </main>
</JoinClient>
```

**No `<SectionTransition>` between Hero and Form.** D-20 noted "Abstand Hero→Form via `<SectionTransition variant="soft-fade" />`", but that intent is superseded by D-19's `min-h-[60vh]` rationale — a transition gap would push the form below the viewport fold on desktop, directly contradicting the core requirement that the form be visible without scroll. D-19 takes precedence.

**No `<SectionTransition>` after form before Footer.** The form section closes the page — it is itself the final CTA cluster (analog to AGENTS.md rule: "Zwischen Final-CTA und letzter Section keine SectionTransition").

### 4. Submit Button

**Note on button label size:** `text-[14px]` = `--fs-button` = `--fs-small`. The submit button label and success CTA label render in Geist Mono at 14px — this is a button-specific size documented in DS §B as `--fs-button`. Both are assigned to **Type Scale role #4 (Micro / `--fs-micro` 11px)** for eyebrow/label pattern. The button label is the one place where the Geist Mono role renders at 14px per the DS button standard (`--fs-button`). Executor must use `text-[14px]` / `var(--fs-button)` on the button label only, not as a general text size.

**States:**
- **Idle:** Label = `"Kostenlos beitreten"` (D-09 single primary CTA, Simon §12 CTA table: "Kostenlos beitreten" for join from landing; here on the join form itself — contextually correct)
- **Loading:** Inline spinner (16px, `animate-spin`) + label = `"Einen Moment…"` (VOICE.md — Loading)
- **Disabled:** `opacity-60`, `cursor-not-allowed`
- **Hover (not disabled):** `boxShadow: "0 0 20px var(--accent-glow)"`, `transform: "scale(1.03)"` (matches partner-contact-form.tsx pattern exactly)

## Motion Contract

**Library:** `motion/react` (Framer Motion API, already in project). All animations gated on `useReducedMotion()`.

### Hero Entry Animation
```js
initial={prefersReducedMotion ? false : { opacity: 0, y: 16 }}
animate={prefersReducedMotion ? {} : { opacity: 1, y: 0 }}
transition={{ duration: 0.7, ease: "easeOut" }}
```
Identical to /about and /partner heroes — visual consistency.

### Form-to-Success Swap (D-22)
Uses `<AnimatePresence mode="wait">` — form exits before success enters.

**Form exit:**
```js
exit={{ opacity: 0, y: -12 }}
transition={{ duration: 0.2, ease: "easeIn" }}
```

**Success entry:**
```js
initial={{ opacity: 0, y: 12 }}
animate={{ opacity: 1, y: 0 }}
transition={{ duration: 0.3, ease: "easeOut", delay: 0.05 }}
```

### Benefit Row Stagger (optional enhancement)
If executor has bandwidth: stagger 3 benefit items with 80ms delay between each, `initial={{ opacity: 0, y: 8 }}` → `animate={{ opacity: 1, y: 0 }}`, `duration: 0.4`. Parent animation starts after hero entry completes (~700ms). Reduced motion: static, no stagger.

### Combobox Dropdown Open/Close
```js
initial={{ opacity: 0, y: -4 }}
animate={{ opacity: 1, y: 0 }}
exit={{ opacity: 0, y: -4 }}
transition={{ duration: 0.15, ease: "easeOut" }}
```

## Accessibility Contract

### ARIA Structure
- `<main id="main-content">` + skip-link from layout (already in project)
- Hero section: `aria-labelledby="join-hero-heading"`, `data-section="join-hero"`
- Form section: `aria-label="Beitrittsformular"`, `data-section="join-form"`
- All inputs: unique `id` via `useId()` + matching `htmlFor` on labels
- Required fields: `aria-required="true"` + `required` attribute
- Invalid fields: `aria-invalid={!!errors.fieldName}`
- Field error descriptions: `aria-describedby={errors.fieldName ? id : undefined}`
- Error messages: `role="alert"` (immediate) on inline field errors; `aria-live="polite"` on server error banner
- Combobox: `role="combobox"`, `aria-autocomplete="list"`, `aria-expanded`, `aria-controls={listboxId}` on input; `role="listbox"` on dropdown; `role="option"` + `aria-selected` on items
- Checkboxes: visually custom-styled, but remain native `<input type="checkbox">` for a11y (no `role="checkbox"` needed)

### Focus Management
- On Success-Swap: `focus()` programmatically on success headline `<p>` or `<h2>` (add `tabIndex={-1}` to receive programmatic focus)
- On Combobox selection: return focus to input, close dropdown
- On form error (client-side): focus first invalid field
- Tab order: top-to-bottom, form fields → checkboxes → submit button

### Focus Visible Styles
```css
/* From brand/DESIGN.md §C Interaction States */
focus-visible:outline: 2px solid var(--text)
focus-visible:outline-offset: 2px
```
Never brand-colored focus rings (would collide with accent on submit button).

Input focus state: `focus-visible:border-[var(--border-accent)]` (DS token, existing pattern from partner-contact-form.tsx).

### Keyboard Navigation
- Combobox: ArrowDown/ArrowUp cycles options, Enter selects, Escape closes, Tab closes and moves to next field
- Checkboxes: Space toggles
- Submit: Enter from any field submits (native form behavior)
- All interactive elements reachable by Tab in logical order

### Touch Targets
All interactive elements: `min-h-[44px]` (WCAG 2.5.5, DS §D: "Touch targets: min-w-[44px] min-h-[44px]"). This applies to:
- All `<input>` fields
- Checkbox wrapper (not just the 20px box — wrapper div has `min-h-[44px]`)
- Submit button
- Success CTA button
- Secondary link (add `py-3` to meet target)

## CSP / Nonce Pattern (LEARNINGS.md Pflicht)

**Server Component** `app/join/page.tsx`:
```ts
import { headers } from "next/headers"

export default async function JoinPage() {
  const nonce = (await headers()).get("x-nonce") ?? ""
  return <JoinClient nonce={nonce} />
}
```

**Client Wrapper** `components/join-client.tsx`:
```tsx
'use client'
export function JoinClient({ nonce }: { nonce: string }) {
  return (
    <MotionConfig nonce={nonce}>
      <Header />
      <main id="main-content" className="min-h-screen pt-20">
        <JoinHeroSection />
        <JoinFormSection />
      </main>
      <Footer />
    </MotionConfig>
  )
}
```

No `border-b border-border` on any section. No `export const dynamic` needed — `await headers()` in page.tsx already forces dynamic rendering.
