<task type="auto">
  <name>Task 1: Kritische A11y-Issues fixen</name>
  <files>apps/website/**, apps/tools-app/**</files>
  <action>
Basierend auf LIGHTHOUSE-AUDIT.md die identifizierten Issues fixen.

Typische A11y-Fixes (anwenden wo relevant):

1. **Fehlende Alt-Texte bei Bildern:**
   - Alle `<img>` Tags pruefen
   - Dekorative Bilder: `alt=""`
   - Informative Bilder: beschreibender Alt-Text

2. **Kontrast-Issues:**
   - Text-Kontrast gegen Hintergrund pruefen (4.5:1 fuer normalen Text, 3:1 fuer grossen Text)
   - CSS-Variablen in base.css anpassen falls noetig

3. **Fehlende Form-Labels:**
   - Alle `<input>`, `<select>`, `<textarea>` brauchen zugehoerige `<label>`
   - Oder `aria-label` / `aria-labelledby`

4. **Fehlende Landmark-Roles:**
   - `<main>`, `<nav>`, `<header>`, `<footer>` Tags pruefen
   - `role` Attribute nur wo semantische Tags nicht moeglich

5. **Keyboard Navigation:**
   - Alle interaktiven Elemente mit Tab erreichbar
   - Focus-States sichtbar (bereits in globals.css vorhanden)
   - Skip-Link vorhanden (bereits in website globals.css)

6. **ARIA-Attribute:**
   - `aria-label` fuer Icon-Only-Buttons
   - `aria-expanded` fuer Dropdowns
   - `aria-current="page"` fuer aktive Navigation

Jede Aenderung dokumentieren: Was war das Issue, wie wurde es gefixt.
